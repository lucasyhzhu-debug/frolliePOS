## Problem (grounded in code)

The POS ships shadcn's stock **light** theme and has never adopted the Frollie/Lucas phthalo-dark canvas the design system was built around. Three concrete facts from the current code:

1. **No theme is ever mounted.** `src/index.css` defines a `:root` (near-white: `--background: oklch(0.99 0 0)`) and a `.dark` block, but **nothing ever adds the `.dark` class** — verified: zero `classList`/`data-theme`/theme-provider references in `src/` (only the literal `.dark {` selector in `index.css` and one `dark:bg-amber-950` utility in `sale/voucher-reject-banner.tsx`). So the app always renders the light `:root`; the entire `.dark` block is dead code today.
2. **~60 design tokens, ~27 of them dead.** `src/index.css @theme` defines `--color-frollie-50..950`, semantic (`success/warning/error/info`), role (`admin/manager/staff/kitchen`), channel (`gofood/grabfood/k3mart`), station (`production/boxing/stickering/packing` × 4 each = 16), and kitchen (`bg/border/counter`) tokens. The **station (16), channel (8 = gofood/grabfood/k3mart families), and kitchen (3) tokens are referenced only inside `src/components/ui/badge.tsx` variant definitions** (`gofood`/`grabfood`/`k3mart` variants) — and **no component renders those badge variants** (to be re-confirmed in the plan). They are kitchen/Frollie-Pro concepts with no POS surface.
3. **Routes bypass tokens with raw Tailwind palette.** 36 raw-palette occurrences across 15 files (`bg-amber-50 text-amber-800` recount/recovery banners in `home.tsx`, `text-teal-*`, etc.) — these don't respond to the theme and will look wrong on a dark canvas.

Separately, Phase 0 (#8) deliberately left **dead animation classes** (`animate-in`, `zoom-in-95`, `slide-in-from-*`) in `dialog/dropdown-menu/popover/select/tooltip` — no animation package is installed, so they are no-ops. The roadmap coupled the install/strip decision to this phase.

The home screen's **top row is dead space** (a version label + 2 status chips), while the single most-frequent action — **New sale** — is one equal-sized tile among ten. Manager-only tiles render **greyed-out with a "mgr only" badge** for staff instead of being hidden (`home.tsx:119-133`).

## Locked decisions (from roadmap brainstorm + this session's design review)

1. **Direction = phthalo-DARK by default** (roadmap-locked). Port the full dark canvas from `frollie-pos design files/lucas-frollie-design-system/project/colors_and_type.css`: `#102821` paper, `#163630` card, `#1E4740` elevated, `#F1E9D8` warm ink, teal `--primary`, **citrus `#F9A84A` accent** (the spark).
2. **Theme mount = permanent `.dark` on `<html>`** (this session): put phthalo-dark in the `.dark` block, add `class="dark"` to `index.html`. `:root` is retuned to an **enriched-light-with-phthalo-accents** fallback. **HARD GATE (roadmap-locked):** real-booth-tablet readability/glare check under mall lighting before app-wide rollout; if it washes out, the fallback is **removing the one `class="dark"` attribute**. Pinned in ADR-047.
3. **Prune the ~27 dead station/channel/kitchen tokens** (this session) — paired with deleting their dead `badge.tsx` variants. Keep role + semantic tokens (manager/staff badges are live in the app).
4. **Design-approval gate = throwaway `design-mock.html` first** (this session): build a static HTML mock of the 3 surfaces + motion feel, get Lucas's sign-off (chrome-devtools tablet screenshots), **then** implement in React. The mock is execution Task 1 with a hard STOP.
5. **Absorb #4 + #5 into this PR** (this session): the home is redesigned here anyway; laying it out three times (#2 visuals → #4 declutter → #5 lock icon) is the exact churn the roadmap warned against (*"home.tsx region — #4, #5, #6 all edit the same area"*). #6 (handoff flow, net-new backend) stays its own phase.
6. **Hero New Sale** (this session): the New-sale action becomes a hero CTA occupying ~half the screen — it's the highest-frequency tap (Fitts's Law).
7. **Motion = engaging but fast** (this session): `tw-animate-css` for the 5 Radix primitives + Framer Motion micro-interactions on the redesigned surfaces. POS motion is fast/tactile/confirming, never decorative delay; honor `prefers-reduced-motion`.
8. **Top app-bar right cluster = Printer + ConnDot only** (this session); Lock icon on the left. No manager/history shortcut in the bar (manager access stays a tile).

## Approach

### A. ADR-047 — pin the direction

New `docs/ADR/047-phthalo-dark-design-system.md`:
- **Decision:** phthalo-dark is the POS default, mounted via a permanent `.dark` class on `<html>`.
- **Mechanism:** phthalo palette lives in `.dark`; `:root` retained as the enriched-light glare fallback.
- **HARD GATE:** real-booth-tablet readability check under mall lighting before declaring rollout done; documented fallback = remove `class="dark"`.
- **Token pruning:** records that station/channel/kitchen tokens + their badge variants were removed (Frollie-Pro kitchen concepts, no POS surface).
- Cross-refs the DS source file and supersedes nothing (additive); notes the citrus accent addition.

### B. Tokens — `src/index.css`

- **`.dark` block → phthalo canvas** (the shipping default), mapping the DS hex to shadcn's oklch/hsl variables: `--background` ← `#102821`, `--card`/`--popover` ← `#163630`, elevated surfaces ← `#1E4740`, `--foreground`/`--card-foreground` ← `#F1E9D8`, `--muted-foreground` ← `#8E8878`, `--border`/`--input` ← `rgba(241,233,216,0.09–0.18)`, `--primary` ← teal (lifted for dark, ~`#14B8A6`), `--ring` ← teal, `--destructive` ← dark-tuned red.
- **New `--accent`/citrus tokens:** add citrus `#F9A84A` (+ hover/soft) as a usable accent token so the spark is available to primitives and surfaces.
- **`:root` → enriched-light fallback:** retune the near-white `:root` toward warm-paper + phthalo accents (coherent with the dark default) so the glare fallback isn't a jarring stock-white theme.
- **Prune** the dead `@theme` tokens: all `--color-station-*` (16), `--color-gofood*`/`--color-grabfood*`/`--color-k3mart*` (8), `--color-kitchen-*` (3). Keep `--color-frollie-50..950`, `--color-success/warning/error/info(-bg)`, `--color-role-*` (used by live badge variants).
- Keep the existing motion tokens (`--ease-*`, `--dur-*`) — Framer Motion + tw-animate-css consume them.

### C. Animations

- Install **`tw-animate-css`** (dev dep, Tailwind-4 native), `@import "tw-animate-css";` in `index.css`. Revives the already-present enter/exit classes on `dialog/dropdown-menu/popover/select/tooltip` — zero markup change.
- Wire **Framer Motion** (already in deps, imported by zero files) for micro-interactions on the redesigned surfaces only (see E). Provide a `prefers-reduced-motion` guard (a small shared helper or per-component `useReducedMotion`).

### D. Primitives — `card.tsx`, `button.tsx`, `badge.tsx`

- **Card:** elevation via the paper-2/paper-3 surfaces + dark-tuned shadow; keep the `Card*` sub-component API unchanged (consumers untouched).
- **Button:** add a tactile `active:scale-[0.97]` press + (optional) primary-gradient treatment via a variant or base class; keep the `variant`/`size` API stable.
- **Badge:** dark-tune the role + semantic variants; **delete** the `gofood`/`grabfood`/`k3mart` variants (paired with the token prune). Verify no consumer references them before deleting.
- The lift propagates by token, so routes not in the redesign list still inherit the dark canvas + enriched primitives.

### E. Three surfaces (laid out once)

**Home (`src/routes/home.tsx`, + `home.test.tsx`)** — absorbs #4 + #5:
- **Top app-bar:** Lock icon (lucide `Lock`) on the left → `navigate("/lock")` (**#5**); brand "Frollie · {name}" + version/catalog line; right cluster = `PrinterSheet` + `ConnDot` (existing components).
- **Hero New Sale:** a large primary CTA (~half the viewport height) replacing the equal-sized `sale` tile; remains a `Link to="/sale"`.
- **Tiles:** redesigned grouped cards with a **reserved photo/initials slot** (so #3 product photos doesn't re-lay-out). **Hide** `mgrOnly` tiles when `!isManager` and **drop empty groups** so the MANAGER header vanishes for staff; delete the disabled/`Badge "mgr only"` branch (**#4**). Move Settlements (`id:"sett"`) to the `mgr` group with `mgrOnly:true` (**#4**).
- **Remove** the redundant bottom Lock button (the footer `<Button>Lock</Button>`) — Lock now lives in the app-bar (**#5**). Keep the bottom New-sale CTA only if the hero doesn't already cover it (hero replaces it).
- Recount/recovery banners: swap `bg-amber-50 text-amber-800` for semantic warning tokens (dark-safe).
- **Tests:** manager renders mgr tiles + Settlements in mgr group; staff does NOT render mgr tiles, MANAGER header absent, Settlements absent; Lock icon present in app-bar; bottom Lock button gone.

**Sale (`src/routes/sale/index.tsx` + cart):**
- Product grid + cart redesigned with real hierarchy on the dark canvas; semantic tokens for any raw palette.
- Framer Motion: tile **tap-to-cart pop** + cart-badge spring (overshoot), cart-line add/remove layout reflow, grid stagger on mount.

**Login/PIN (`src/routes/login.tsx` shell, `NumericKeypad.tsx` visual):**
- **Visual shell only** — dark canvas, brand treatment, keypad/dot styling via tokens. **No `PinEntry` submit/spinner/inline-error logic** (owned by #7/#11 — do not touch that logic to avoid the documented collision). Keypad press-state styling is fine as pure CSS, but leave the submit flow alone.

**Charge-success (`src/routes/sale/charge-success.tsx`):**
- Signature **payment-success celebration** (checkmark draw + teal/citrus lift) — the emotional peak of the sale. Pure Framer Motion + tokens.

### F. Raw-palette sweep

Replace raw Tailwind palette usages with semantic tokens across the 15 files surfaced by grep (home banners, `sale/*`, `mgr/receipt`, `stock/*`, `approve/*`, etc.). Scope to dark-safety: anything that sets a literal light background/foreground that breaks on dark. **Leave `id-ID` currency formatting in `src/lib/format.ts` untouched** (currency, not copy — i18n is #1).

### G. Approval gate (execution Task 1)

Build throwaway `design-mock.html` (gitignored or deleted before merge) rendering the 3 surfaces + motion feel with the new tokens. Screenshot at tablet viewport via chrome-devtools-mcp (`emulate`/`resize_page`) — this also serves the glare-gate **dev** pass. **STOP for Lucas's sign-off** before propagating tokens app-wide.

## Risk

Medium — broad surface (tokens touch the whole app by design) but mechanically additive and reversible. Biggest risks: (a) **dark readability** (mitigated by the glare HARD GATE + one-line fallback); (b) **token prune over-reach** (mitigated by re-confirming zero badge-variant usage in the plan before deleting); (c) **collision with #7/#11** in `login.tsx`/`NumericKeypad.tsx` (mitigated by restricting #2 to pure visual/CSS there, no logic). No schema, no backend, no deploy-skew surface.
