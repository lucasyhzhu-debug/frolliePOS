# v0.5.7 ‚Äî Orphaned-function wiring (design spec)

**Date:** 2026-06-05
**Phase:** v0.5.7
**Source handoff:** `.claude/handoff/execute_v0.5.7-orphan-wiring.md`
**Scope:** Wire three public Convex functions that exist + are tested but have **zero UI callers** ‚Äî the same "backend exists, no door" gap v0.5.6 closed. Mostly frontend; **one small backend change** (Part A actor-label pre-derivation, decided below).

## Problem

A 2026-06-03 audit of every `convex/**/{public,actions}.ts` export against `src/` `api.` callers found three genuinely orphaned public functions:

| Part | Function | Location | Today |
|---|---|---|---|
| A | `audit.public.list` | `convex/audit/public.ts:7` | manager-gated audit query, no viewer |
| B | `transactions.public.listRecentAwaitingPayment` | `convex/transactions/public.ts:392` | recovery query, no caller (hook never built) |
| C | `approvals.public.cancelPendingRequest` | `convex/approvals/public.ts:509` | cancel mutation, no UI affordance |

(`inventory.public.getStockLevels` + `vouchers.public.getActiveVouchers` looked orphaned but are consumed cross-module by `catalog.public.catalog` per ADR-034 ‚Äî **not gaps, left alone.**)

## Verified facts (read against real code, 2026-06-05)

1. **`audit.public.list` returns raw `audit_log` rows** via `auditListHandler` (`convex/audit/internal.ts:57`). Rows carry `actor_id` (`Id<"staff"> | "system"`), `action`, `entity_type`, `entity_id?`, `source`, `reason?`, `mgr_approver_id?`, `created_at` ‚Äî **no actor name**. Args `{ sessionId, limit?, action? }`; `limit` clamped to `min(limit ?? 100, 500)`; `action` filter uses `by_action_date` index; default order `desc`.
2. **`_listStaffNames_internal` exists** at `convex/auth/internal.ts:469` and is **already used this exact way** by `convex/transactions/internal.ts:593` to build a staff-id‚Üíname map for reporting labels. This is the established "BE pre-derives labels" pattern (v0.5.3a lesson).
3. **`listRecentAwaitingPayment`** (`convex/transactions/public.ts:392`) ‚Äî args `{ sessionId }`, returns `Doc<"pos_transactions">[]` in `awaiting_payment` from the last 5 min via `by_status_created`, `[]` on invalid session. Booth-wide (not per-staff), correct for a single shared device. Rows come back ascending `created_at` (index order) ‚Äî "latest" = max `created_at`.
4. **`/sale/charge/:txnId` resumes cleanly.** `charge.tsx` reads `:txnId` and subscribes reactively via `useXenditPayment`; an existing awaiting txn with a live invoice lands in `phase==="showing"` and re-renders its QR. The init-create effect skips creation when an invoice already exists for the selected method (`charge.tsx:175`). **No resume guard needed.** (Edge: charge defaults to the QRIS tab; if the awaiting invoice was BCA_VA the existing tab-swap logic re-mints a QRIS invoice ‚Äî pre-existing single-active-invoice behavior, ADR-014, acceptable.)
5. **`cancelPendingRequest`** (`convex/approvals/public.ts:509`) ‚Äî `withIdempotency` mutation, args `{ sessionId, requestId, reason?, idempotencyKey }` ‚Üí `{ denied: boolean }`, `requireManagerSession`-gated, audited `source: "booth_inline"`, records `cancelled_by_manager_id`. **Manager-session gated** ‚Äî caller must be a manager session.
6. **`ApprovalPending`** (`src/components/pos/ApprovalPending.tsx`) ‚Äî props `{ requestId, successMessage?, onResolved?, onDenied?, onExpired? }`. The `pending` branch (lines 49-58) renders a spinner + "Waiting‚Ä¶" text with **no button**. `onDenied`/`onExpired` already render symmetric "Retry" buttons.
7. **Part C applies to only TWO hosts, not three.** `sale/charge.tsx` (`approvalRequestId` state, renders `<ApprovalPending>` at ~`:591`) and `refund/detail.tsx` (`approvalRequestId` state, renders `<ApprovalPending>` at ~`:284`) both hold a pending request. **`mgr/spoilage.tsx` does NOT** ‚Äî its Telegram path (`handleRequest`, `:195`) is fire-and-forget: it toasts "Sent to managers via Telegram" and calls `resetForm()`, never rendering `ApprovalPending` nor holding a `requestId`. **The handoff's claim that spoilage needs `onCancel` is wrong** ‚Äî there is no pending UI there to cancel.
8. **Manager-gate pattern** (`mgr/home.tsx:38`, `mgr/spoilage.tsx:79`): `session.status !== "active" || session.staff.role !== "manager"` ‚Üí `<Navigate to="/" replace />`. `refunds-pending.tsx` uses an inline "Manager access required" card instead ‚Äî both are in use; the `<Navigate>` form is the v0.5.6 convention for new spokes.
9. **Router** (`src/router.tsx`): `lazy()` import + a child route object under the `/` `RootLayout` group. `NAV_CARDS` on `mgr/home.tsx:13` is a flat array of `{ to, label, hint, glyph }`.

## Decision: Part A actor labels ‚Äî BE pre-derive (user-approved 2026-06-05)

`audit.public.list` will be **extended to attach `actor_name` server-side** via `_listStaffNames_internal`, mapping each row's `actor_id` (`"system"` ‚Üí `"System"`, unknown id ‚Üí fallback to the raw id string). Rationale: matches the v0.5.3a "BE pre-derives labels when it has all inputs" lesson and the existing `transactions/internal.ts` precedent; avoids a client-side second query + id‚Üíname join. **Consequence: this phase is not strictly frontend-only ‚Äî it includes one backend query change + its test.** `mgr_approver_id` is left as-is (not surfaced in v0.5.7).

## Design

### Part A ‚Äî Audit-log viewer (`/mgr/audit`)

- **Backend:** extend `audit.public.list` handler to map rows ‚Üí `{ ...row, actor_name }`. Fetch the staff-name map once via `ctx.runQuery(internal.auth.internal._listStaffNames_internal, {})` (cross-module read per ADR-034), then label. Return type changes from `Doc<"audit_log">[]` to `(Doc<"audit_log"> & { actor_name: string })[]`. Keep `_list_internal` untouched (no label ‚Äî tests/server callers don't need it). Add a test asserting `actor_name` resolves for a real staff actor and `"System"` for `"system"`.
- **Frontend:** new `src/routes/mgr/audit.tsx` ‚Äî manager-gated spoke (outer role-guard ‚Üí `<Navigate to="/" replace />`, inner data component taking `sessionId`), mirroring `mgr/refunds-pending.tsx` + `mgr/stock.tsx`. `SpokeLayout title="Audit log" backTo="/mgr"`. Reverse-chron `Card` rows: `fmtDate`/`fmtTime` timestamp, action verb (mono), `actor_name`, `entity_type` + `entity_id`, a `source` badge, and `reason` (line-clamped) when present. ADR-007 ‚Äî audit is append-only/read-only, **no row actions.**
  - **Action filter:** a controlled text `Input` (or `Select` over distinct verbs from the loaded rows); when non-empty, passes `action` to the query. Keep it simple ‚Äî text input feeding `action` arg, debounced is optional.
  - **Load more:** local `limit` state (default 100), a "Load more" button bumps it by 100 up to the server cap (500). Reactive query re-runs on `limit` change.
- **Nav:** add a 10th `NAV_CARD` to `mgr/home.tsx` (`{ to: "/mgr/audit", label: "Audit log", hint: "Append-only activity trail", glyph: "‚ù°" }` or similar) + lazy route `mgr/audit` in `src/router.tsx`.

### Part B ‚Äî Awaiting-payment recovery banner (`home.tsx`)

- **Hook:** new `src/hooks/useAwaitingPaymentRecovery.ts`, shaped after `useRecountNudge.ts`. Runs `useQuery(api.transactions.public.listRecentAwaitingPayment, sessionId ? { sessionId } : "skip")`. Returns `{ count: number; latest: Doc<"pos_transactions"> | null }` ‚Äî `count = list.length`, `latest =` the row with max `created_at` (or `null` while loading / empty). Returns `{ count: 0, latest: null }` while `undefined` (loading) so the banner never flashes.
- **Banner:** in `home.tsx`, next to the existing recount nudge (`home.tsx:86-93`), render an amber `Link`-style banner when `count > 0`: e.g. `"{count} pembayaran belum selesai ‚Äî ketuk untuk lanjutkan"`. Tapping `navigate`s to `/sale/charge/${latest._id}`. Read-only, **no backend change.**

### Part C ‚Äî Cancel a pending approval (`ApprovalPending`, refund host only)

**DECIDED (staffreview 2026-06-05, user):** `cancelPendingRequest` is `requireManagerSession`-gated and **throws `MANAGER_ONLY`** for non-managers (`convex/auth/sessions.ts:24-31`; called in both the handler `:527` and `authCheck` `:539`). It was built as an on-booth **manager** cancel (its docstring: *"from the on-booth manager panel"*), NOT a staff-requester self-cancel. Since the off-booth Telegram flow is taken precisely when no manager is at the booth, the requester is usually a non-manager ‚Äî so the button must be **gated to manager sessions**, and the phase wires **refund only** (charge's cancel is near-redundant with the existing sale-abandon cascade ‚Äî see Out of scope).

- **Component:** add optional `onCancel?: () => void` to `ApprovalPending`. When provided, render a "Batalkan permintaan" ghost/outline button **inside the `pending` branch only** (it makes no sense after a terminal status). Symmetric to the existing `onDenied`/`onExpired` Retry buttons. The component itself is role-agnostic ‚Äî the host decides whether to pass `onCancel`.
- **Host (1): `refund/detail.tsx`** ‚Äî pass `onCancel` to its `<ApprovalPending>` (~`:284`) **only when `session.staff.role === "manager"`** (so non-managers never see a button that would throw `MANAGER_ONLY`). `onCancel` calls `cancelPendingRequest({ sessionId, requestId: approvalRequestId, idempotencyKey: crypto.randomUUID() })`; on success `setApprovalRequestId(null)` **and** `clearIntent("refund-telegram:" + txnId)` (so a retry mints a fresh request, consistent with the existing `handleApprovalDenied`/`handleApprovalExpired` at `:244-253`). Map errors via `toast.error` (reuse the local `mapErr`).
- **Non-manager exit paths (unchanged, documented):** a non-manager requester who needs to abort relies on the existing exits ‚Äî the 60-min token TTL expiry, manager denial via Telegram, or (for charge) sale abandonment which already cascade-denies the pending manual-payment request (`_cancelPendingManualPaymentForTxn_internal`, `transactions/public.ts:420-421`). True staff self-cancel is a deferred backend feature, **out of v0.5.7 scope.**
- **`/wait/:requestId`** (`src/routes/wait.tsx`) is a Stub; the real pending UX is the inline `ApprovalPending`, which is why cancel belongs there. Leave the dead `/wait` route as-is (out of scope).

## Out of scope / non-goals

- No schema, migration, or new index.
- No changes to `mgr/spoilage.tsx` (fire-and-forget, no pending UI ‚Äî fact #7).
- **No cancel button on `sale/charge.tsx`** ‚Äî abandoning the sale already cascade-denies the pending manual-payment approval (`_cancelPendingManualPaymentForTxn_internal`), so an explicit manager-only cancel there is near-redundant. `charge.tsx`'s `<ApprovalPending>` is left unchanged (no `onCancel` passed; the new optional prop is simply absent).
- **No staff-requester self-cancel** (would be net-new backend ‚Äî deferred).
- No retiring of the `/wait` stub.
- `mgr_approver_id` not surfaced in the audit viewer.
- No new approval KIND, no Telegram template change.

## Testing

- **Part A:** BE test ‚Äî `actor_name` resolves for a staff actor + `"System"` for `"system"`. FE tests ‚Äî manager renders rows; non-manager redirects (`<Navigate>`); action filter passes `action` arg; "Load more" bumps `limit`.
- **Part B:** banner shows when list non-empty + navigates to latest `_id`; hidden when empty/loading.
- **Part C:** `ApprovalPending` shows the cancel button only in the `pending` branch **and only when `onCancel` is provided** (component-level test). In `refund/detail.tsx`: cancel `onCancel` is passed **only for a manager session** (non-manager ‚Üí no button, asserting it never calls a throwing mutation); clicking calls `cancelPendingRequest` once with a UUID key; on success resets `approvalRequestId` to `null` and clears the telegram intent.
- Harness: `vi.mock("convex/react")` dispatching `useQuery`/`useMutation`/`useAction` by `getFunctionName` + `MemoryRouter` (see `src/routes/history/__tests__/$txnId.test.tsx`).
