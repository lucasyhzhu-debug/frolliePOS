# v2.0 Owner Auth Plane (Telegram-OTP cockpit login) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the owner authentication plane — `/cockpit/login` (account → Telegram-OTP → durable cockpit session), one-time `/start <token>` Telegram binding, durable `kind:"cockpit"` sessions with idle-timeout + remembered-device quick-PIN, and the new outlet-unscoped `owner` role — distinct from booth PIN auth. Auth machinery ONLY (not the cockpit screens).

**Architecture:** Owner auth is a third verb — "OTP authorises MANAGE" (ADR-052), extending ADR-029 (token=VIEW, PIN=ACT). A 6-digit OTP delivered to the owner's **private** Telegram DM mints a durable cockpit session on `staff_sessions` discriminated by `kind`. The cockpit plane is isolated from booth lockout (`pos_auth_attempts`) and from outlet scoping (cockpit sessions carry no `outlet_id`). Reuses the `loginWithPin` action→commit-mutation pattern, `mintUrlSafeToken`, argon2id-in-action (ADR-004), `sha256Hex` token hashing, and the existing `sendTemplate` `chatIdOverride` DM path.

**Tech Stack:** Convex 1.31.7 (TypeScript), `hash-wasm` (argon2id), React 19 + Vite, `convex-test` + vitest (edge-runtime), Telegram Bot API.

**Spec:** `docs/superpowers/specs/2026-06-21-owner-auth-plane-design.md`
**ADR:** `docs/ADR/052-owner-auth-telegram-otp.md`
**Review:** `docs/reviews/staffreview-owner-auth-plane-spec-2026-06-22.md`

Where this plan **amends** the Spec-1 plan, it is marked `[SPEC-1 AMENDMENT]` — fold those into Spec-1 execution if it has not enforced yet, else re-apply here.
- **Two required Spec-1 amendments** (Task 1): (a) `staff_sessions.outlet_id` MUST stay `v.optional` (cockpit sessions are outlet-less) — exclude it from the Spec-1 Task-12 required-flip; (b) `owner_auth_otp` / `owner_auth_bindings` / `owner_auth_attempts` go in the Spec-1 fence's **business-level exclusion** list, NOT `OUTLET_SCOPED`.
- **argon2id in `"use node"` actions, NEVER mutations** (ADR-004). Hash via `argon2id` from `hash-wasm` with `ARGON2_PARAMS` (`convex/auth/actions.ts:11`). Commit mutations only store/compare pre-hashed strings.
- **High-entropy tokens** (bind / remember-device, 32-byte) hash with the **V8-safe async** `sha256Hex` from **`convex/lib/sha256.ts`** (Web Crypto — `await` it). **Do NOT use `convex/lib/tokenHash.ts`** — that one is `"use node"` (node:crypto) and cannot be imported into edge-runtime `convex-test` files or called from a V8 webhook action (PL-1). Look up by exact `token_hash` (`by_token_hash`); a hash-equality lookup is already constant-time-safe (no secret branch). **Low-entropy secrets** (6-digit OTP, 4-6 digit quick-PIN) hash with argon2id.
- **Server time wins** (ADR-031): every `*_at`, `expires_at`, TTL, idle-timeout from `Date.now()` **inside the handler**. OTP code is server-minted; client never supplies it.
- **Public mutations/actions require `idempotencyKey` + `withIdempotency` + `authCheck`** (ADR-013). **Distinct keys per intent**: an action→mutation chain (e.g. `verifyOwnerOtp` → `_cockpitLoginCommit_internal`) MUST use distinct keys (derive a `:commit` suffix) — `pos_idempotency` is keyed by `key` alone (MEMORY: idempotency shared-key collision).
- **Audit isolation (SEC-07):** OTP / quick-PIN misses are audited but use their OWN counters (`owner_auth_attempts`, per-binding lockout) — they NEVER write `pos_auth_attempts`.
- **Never log the OTP code or any PIN** (verifyPin.ts convention). `sendTemplate.logOutbound` must redact `owner_otp`.
- **`convex/lib/**` is V8-safe** (no `"use node"`).
- **Audit `source` for owner verbs is `system`** (bot-mediated) — NOT `telegram_approval` (reserved for PIN approval flows). `audit_log.action` is a free `v.string()` — no enum to edit.
- **Tests:** `npx vitest run <file>` per task; full gate `npm run typecheck && npm run lint && npx vitest run` before merge.

```ts
export const COCKPIT_IDLE_MS = 30 * 60 * 1000; // 30 min sliding

export async function requireCockpitSession(
  ctx: QueryCtx | MutationCtx,
  sessionId: Id<"staff_sessions">,
): Promise<{ staffId: Id<"staff">; deviceId: string }> {
  const s = await ctx.db.get(sessionId);
  if (!s || s.ended_at != null) throw new Error("NO_SESSION");
  if ((s.kind ?? "booth") !== "cockpit") throw new Error("NOT_COCKPIT_SESSION");
  const staff = await ctx.db.get(s.staff_id);
  if (!staff || !staff.active || staff.role !== "owner") throw new Error("NO_SESSION");
  if (s.last_active_at != null && Date.now() - s.last_active_at > COCKPIT_IDLE_MS) {
    throw new Error("SESSION_IDLE_TIMEOUT");
  }
  return { staffId: s.staff_id, deviceId: s.device_id };
}

export const touchCockpitSession = mutation({
  args: { idempotencyKey: v.string(), sessionId: v.id("staff_sessions") },
  handler: withIdempotency<{ idempotencyKey: string; sessionId: Id<"staff_sessions"> }, null>(
    "auth.touchCockpitSession",
    async (ctx, args) => {
      const s = await ctx.db.get(args.sessionId);
      if (!s || s.ended_at != null || (s.kind ?? "booth") !== "cockpit") return null;
      await ctx.db.patch(args.sessionId, { last_active_at: Date.now() });
      return null;
    },
    { authCheck: async (ctx, args) => { await requireCockpitSession(ctx, args.sessionId); } },
  ),
});

export const logoutCockpit = mutation({
  args: { idempotencyKey: v.string(), sessionId: v.id("staff_sessions") },
  handler: withIdempotency<{ idempotencyKey: string; sessionId: Id<"staff_sessions"> }, null>(
    "auth.logoutCockpit",
    async (ctx, args) => {
      const s = await ctx.db.get(args.sessionId);
      if (!s || s.ended_at != null) return null;
      await ctx.db.patch(args.sessionId, { ended_at: Date.now(), end_reason: "manual_lock" });
      await logAudit(ctx, {
        actor_id: s.staff_id, action: "owner.logout",
        entity_type: "staff_session", entity_id: args.sessionId,
        source: "system", device_id: s.device_id,
      });
      return null;
    },
    { authCheck: async () => {} }, // logout is idempotent — graceful no-op like auth.logout
  ),
});
```

  name: string;

export const _redeemBinding_internal = internalMutation({
  args: { tokenHash: v.string(), fromId: v.number(), chatType: v.string() },
  handler: async (ctx, { tokenHash, fromId, chatType }) => {
    if (chatType !== "private") throw new Error("BIND_PRIVATE_ONLY");
    const b = await ctx.db.query("owner_auth_bindings").withIndex("by_token_hash", (q) => q.eq("token_hash", tokenHash)).first();
    if (!b || b.kind !== "telegram_bind" || b.redeemed_at != null || b.expires_at < Date.now()) throw new Error("BIND_INVALID");
    const dup = await ctx.db.query("staff").withIndex("by_telegram_user_id", (q) => q.eq("telegram_user_id", fromId)).first();
    if (dup && dup._id !== b.staff_id) throw new Error("TELEGRAM_ALREADY_BOUND");
    await ctx.db.patch(b._id, { redeemed_at: Date.now() });
    await ctx.db.patch(b.staff_id, { telegram_user_id: fromId });
    await logAudit(ctx, { actor_id: b.staff_id, action: "owner.telegram_bound", entity_type: "staff", entity_id: b.staff_id, source: "system" });
    return { staff_id: b.staff_id };
  },
});
