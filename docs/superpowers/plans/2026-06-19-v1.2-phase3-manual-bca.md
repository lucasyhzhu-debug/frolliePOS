> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Hide the broken BCA VA tab (killing the toast storm by construction) and add a staff-self-confirm manual bank-transfer tender â€” marked `manual_bca`, flagged in the Telegram ticker, and reconciled by an itemized EOD summary.

**Architecture:** QRIS becomes the sole Xendit method. A new staff-session **mutation** funnels manual confirms through the existing `_confirmPaid_internal` with a new `source:"manual_bca"` marker (stamped on `confirmed_via`). Static account config lives in new `pos_settings` fields (manager CRUD, mirror of receipt-config). The marker drives a distinct ticker line and an internal reconciliation query the founders cron (and later #6) consumes.

**Tech Stack:** Convex (TypeScript), React + React Router, convex-test + vitest, Telegram HTML templates.

**Spec:** `docs/superpowers/specs/2026-06-19-v1.2-phase3-manual-bca-design.md`. **Spec staffreview:** `docs/reviews/staffreview-v1.2-phase3-manual-bca-spec-2026-06-19.md`.

## Global Constraints

- **Money = integer rupiah** (ADR-015). `account_number` is an identifier **string** (leading-zero safe), never coerced.
- **Server time** (ADR-031): all `_at` fields set inside Convex fns via `Date.now()`.
- **Public mutations** take `idempotencyKey` + `withIdempotency` + `authCheck` (ADR-013, ESLint-enforced).
- **Audit** (ADR-007): emit `logAudit` from writers. `audit_log.action` is `v.string()` â€” new verbs need **no migration**.
- **Cross-module reads** go through owning-module internals (ADR-034) â€” never `ctx.db` on another module's table.
- **Deploy-skew:** ship backend + frontend in ONE squash PR (atomic build). Do **NOT** remove the `BCA_VA` schema literal or rename any mutationâ†”action.
- **Convex `lib/` is V8-safe** â€” no `Buffer`/node imports in `telegramHtml.ts`.
- **Optional-field filter gotcha:** never `q.eq(optionalField, undefined)`; filter in JS (own MEMORY: `convex-optional-field-filter-gotcha`).
- **Tests REQUIRED on the payment path** (CLAUDE.md #7).
- **Evidence-gate verdict (plan staffreview Â§4.9):** the error-storm remedy is evidence-backed (concrete mechanism: `charge.tsx:173-233` re-fires `createBcaVaCharge` â†’ `XENDIT_VA_FAILED` `xendit.ts:96` â†’ `toast.error` per attempt) and is a **structural removal** (hide the BCA VA tab â†’ QRIS is the only auto-created instrument), not a timing/debounce mitigation â†’ **no Task 0 needed**.

```ts
import { mutation, query } from "../_generated/server";
import { v } from "convex/values";
import { internal } from "../_generated/api";
import type { Id } from "../_generated/dataModel";
import { wibDayWindow } from "../lib/time";
import { withIdempotency } from "../idempotency/internal";
import { requireSession } from "../auth/sessions";

/**
 * v1.2 #10 â€” staff self-confirm manual bank transfer (ADR-036 amendment).
 * NO manager PIN, NO Xendit call. Funnels through _confirmPaid_internal with
 * source="manual_bca" so snapshot pricing + the by_line_and_sku decrement guard
 * hold (rule #17). Then supersedes any live QRIS invoice (I1) so a stale QR
 * can't be double-paid. The EOD/clock-out reconciliation summary is the
 * compensating control for the missing photo gate.
 */
export const confirmManualBcaPayment = mutation({
  args: {
    idempotencyKey: v.string(),
    sessionId: v.id("staff_sessions"),
    txnId: v.id("pos_transactions"),
  },
  handler: withIdempotency<
    { idempotencyKey: string; sessionId: Id<"staff_sessions">; txnId: Id<"pos_transactions"> },
    { confirmed: true; receiptNumber: string }
  >(
    "payments.confirmManualBcaPayment",
    async (ctx, args) => {
      const { staffId } = await requireSession(ctx, args.sessionId);
      await ctx.runMutation(internal.transactions.internal._confirmPaid_internal, {
        txnId: args.txnId,
        source: "manual_bca",
        confirm_staff_id: staffId,
      });
      // Read back: a non-awaiting txn no-ops in the funnel â†’ no receipt â†’ throw
      // rather than report a false success (C4, mirrors _onPaidManual_internal).
      const txn = await ctx.db.get(args.txnId);
      if (!txn || txn.status !== "paid" || !txn.receipt_number) {
        throw new Error("RECEIPT_UNCONFIRMED");
      }
      // I1: supersede the live QRIS invoice ONLY when WE won the race (our manual
      // confirm flipped it). If a webhook already paid (confirmed_via==="webhook"),
      // do NOT cancel the paying invoice â€” just return its receipt (idempotent).
      if (txn.confirmed_via === "manual_bca") {
        await ctx.runMutation(internal.payments.internal._cancelActiveInvoiceForTxn_internal, {
          txnId: args.txnId,
          cancel_reason: "manual_bca_confirmed",
          actor_id: staffId,
          source: "booth_inline",
        });
      }
      return { confirmed: true as const, receiptNumber: txn.receipt_number };
    },
    {
      // ADR-013: gate the cache lookup itself so a same-key replay from an
      // invalid session can't read back {confirmed} without authn.
      authCheck: async (ctx, args) => {
        await requireSession(ctx, args.sessionId);
      },
    },
  ),
});
```

> The existing `getCurrentInvoice` query stays as-is below the new mutation.

```ts
/**
 * v1.2 #10 â€” itemized manual-BCA reconciliation for a WIB day window. The
 * compensating control for staff-self-confirmed transfers (no photo gate):
 * lists every manual_bca paid txn so operators validate each deposit against
 * the real BCA account. Consumed by the founders EOD cron and (later) #6's
 * clock-out sign-off. Reuses by_status_paid_at + a JS filter on confirmed_via
 * (no dedicated index â€” booth volume is tiny). Resilient on a missing staff
 * name (EOD cron must not crash) â†’ "Staff".
 */
export const _manualBcaReconciliation_internal = internalQuery({
  args: { dayStartMs: v.number(), dayEndMs: v.number() },
  handler: async (
    ctx,
    args,
  ): Promise<{
    items: Array<{ paidAt: number; total: number; staffName: string; receiptNumber: string }>;
    count: number;
    totalIdr: number;
  }> => {
    const paid = await ctx.db
      .query("pos_transactions")
      .withIndex("by_status_paid_at", (q) =>
        q.eq("status", "paid").gte("paid_at", args.dayStartMs).lt("paid_at", args.dayEndMs),
      )
      .collect();
    const manual = paid
      .filter((tx) => tx.confirmed_via === "manual_bca")
      .sort((a, b) => a.paid_at! - b.paid_at!); // chronological for reconciliation reading
    const staffNames = await ctx.runQuery(internal.auth.internal._listStaffNames_internal, {});
    const nameById = new Map(staffNames.map((s) => [String(s._id), s.name]));
    const items = manual.map((tx) => ({
      paidAt: tx.paid_at!,
      total: tx.total,
      staffName: nameById.get(String(tx.staff_id)) ?? "Staff",
      receiptNumber: tx.receipt_number ?? "â€”",
    }));
    return { items, count: items.length, totalIdr: items.reduce((s, x) => s + x.total, 0) };
  },
});
```
