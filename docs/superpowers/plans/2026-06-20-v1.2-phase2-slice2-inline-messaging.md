# v1.2 #12 slice 2 — inline messaging Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert the remaining bucket-A sync form-validation toasts to inline `FieldMessage` across settlements / staff / device-activation / receipt, refactor `stock/$skuId` server errors behind a local humanizer, and repair the ADR-048 ESLint fence so it guards these (now i18n-wrapped) files.

**Architecture:** Reuse-only — the slice-1 design system (`FieldMessage`, `useFieldErrors`, ADR-048) carries over verbatim. Each converting file adopts the slice-1 idiom (module-scope `*_FOCUS` map → `applyErrors(prefix, next, FOCUS)` collect-all → per-field `<FieldMessage>` + aria wiring). Legitimate async/precondition toasts use the variable idiom (`const msg = t(...); toast.error(msg)`) so the new `t()`-arg fence doesn't false-positive.

**Tech Stack:** React 19 + TS, Tailwind 4, shadcn/ui, Sonner, vitest + RTL (`@/test-utils` `renderWithLocale`), ESLint flat config.

**Spec:** `docs/superpowers/specs/2026-06-20-v1.2-phase2-slice2-inline-messaging-design.md`
**Spec review:** `docs/reviews/staffreview-v1.2-phase2-slice2-inline-messaging-spec-2026-06-20.md`

## Global Constraints

- **No new primitive / token / variant.** Reuse `FieldMessage` (`error`/`success` only) + `useFieldErrors` exactly.
- **Validation copy goes through `t(...)`** (keys mostly exist; add only `mgrStaff.errorNameLength` + `mgrStaff.errorPinDigits`). en + id parity mandatory.
- **Bucket-C/precondition toasts stay toasts** but via the **variable idiom** (`const msg = t(...); toast.error(msg)`), never a bare `toast.error(t(...))`.
- **Dotted-id convention:** message `id="prefix.field-error"`; reference via `getElementById` only (never `querySelector`).
- **Test mocks:** `vi.mock("@/hooks/useIdempotency", () => ({ useIdempotency: () => "key1", clearIntent: vi.fn() }))` (jsdom-undefined disables submit otherwise); mock `sonner` to assert toast (not) fired.
- Verify gate each task: `npx vitest run <file>` green. Final: `npm run typecheck && npm run lint && npx vitest run src/ && npm run build:fe`.

```tsx
const { errors, clearFieldError, clearErrors, applyErrors } = useFieldErrors();
// module scope:
const X_FOCUS: Record<string, string> = { "prefix.field": "dom-input-id", /* in focus order */ };
// open handler: clearErrors("prefix.");
// submit handler:
const next: Record<string, string> = {};
if (!valid) next["prefix.field"] = t("some.key");
if (applyErrors("prefix.", next, X_FOCUS)) return;
// JSX, after the <Input>:
<Input id="dom-input-id" aria-invalid={!!errors["prefix.field"]}
  aria-describedby={errors["prefix.field"] ? "prefix.field-error" : undefined}
  onChange={(e) => { setX(e.target.value); clearFieldError("prefix.field"); }} />
{errors["prefix.field"] && <FieldMessage id="prefix.field-error">{errors["prefix.field"]}</FieldMessage>}
```
