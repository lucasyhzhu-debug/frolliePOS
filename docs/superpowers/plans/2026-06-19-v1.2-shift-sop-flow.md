# Shift SOP Flow Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the booth opening/closing paper SOPs into guided in-app wizards backed by a booth state machine, so signing in walks staff through start-of-day, "End shift" runs close-or-handover, stock counts land in inventory, and every shift boundary produces an attributed, audited record.

**Architecture:** A new `convex/shifts/` deep module owns a `pos_shift_events` table that is the single source of truth for booth state (CLOSED / OPEN / LOCKED / HANDOVER_PENDING), attendance, the step-completion ledger, and per-shift summaries. State is *derived* from the latest event row (pure helper in `shifts/lib.ts`). Lifecycle writers (mutations + one takeover action) record events and end/begin sessions, reusing existing `recordRecount`, `loginWithPin`, `withIdempotency`, `requireSession`, and `verifyPinOrThrow`. The FE adds a shared wizard (step-rail + count step extracted from the recount route) and forks the login gate on booth state. A per-shift `staff_shift_signoff` Telegram template (distinct from the existing daily `shift_summary`) posts financials to Founders, reusing the `manualBca` renderer fragment.

**Tech Stack:** Convex 1.31.7 (queries/mutations/actions, convex-test + vitest), React 19 + TypeScript + Vite, Tailwind 4 / shadcn, Framer Motion, IDB idempotency.

## Global Constraints

- **Money:** integer rupiah only; format via `src/lib/format.ts` (`rp`). No floats. (CLAUDE.md #14)
- **Server time:** all `_at` fields set inside the function via `Date.now()` (ADR-031). Never client-supplied.
- **Idempotency:** every public mutation takes `idempotencyKey` + wraps `withIdempotency(name, handler, { authCheck })`; the handler re-calls `requireSession`/`requireManagerSession` so authCheck runs before cache lookup (ADR-013/046, rule #20). Actions use `withActionCache(ctx, {key, mutationName}, authCheck, fn)`.
- **Audit:** every state-changing mutation emits `logAudit`; `audit_log.action` is a free `v.string()` (no enum). New verbs: `shift.start_of_day`, `shift.signoff`, `shift.handover_out`, `shift.handover_in`, `shift.manager_takeover`, `shift.lock`, `shift.resume`.
- **V8-safe:** `convex/shifts/lib.ts` and `convex/lib/telegramHtml.ts` must not use `"use node"`.
- **Stock writes:** count steps call the existing `api.inventory.public.recordRecount` (source `"recount"`, ADR-041) — never a new stock-write path.
- **ADR-003:** lock still ENDS the session; LOCKED is a booth-state layer, not a held session.
- **Copy:** Bahasa for staff-facing strings (EN toggle deferred to #1). Extraction-ready (no concatenated sentences).
- **Tokens:** semantic phthalo-dark tokens only (`bg-card`, `text-muted-foreground`, `text-citrus`, `text-error`/`text-success`); guard Framer Motion with `useReducedMotion`.

## Test conventions (applies to every test snippet below)

- Backend tests: `const t = convexTest(schema);` — **single arg**, no `modules`/`test.setup` import (matches `convex/auth/__tests__/list-staff-names.test.ts:8`). The snippets below that show `convexTest(schema, modules)` are shorthand — use the single-arg form.
- Seed `staff` with the full shape incl. `created_at`, casting `as any` per the existing tests: `ctx.db.insert("staff", { name, role, active: true, pin_hash: "x", code, created_at: 0 } as any)`.
- `staff_sessions.end_reason` is a **closed union** (`"manual_lock" | "timeout" | "force_logout" | null`, `convex/auth/schema.ts:27`). Session-end writes reuse it: **`"manual_lock"`** for lock, **`"force_logout"`** for signoff / handover / takeover. No `staff_sessions` schema change.

```typescript
// convex/shifts/lib.ts
import type { Id } from "../_generated/dataModel";

export type ShiftEventType =
  | "start_of_day" | "lock" | "resume"
  | "signoff_close" | "handover_out" | "handover_in" | "manager_takeover";
export type BoothState = "closed" | "open" | "locked" | "handover_pending";
export type LatestEvent = {
  type: ShiftEventType; staff_id: Id<"staff">; created_at: number; shift_started_at: number;
} | null;

const OPEN_TYPES: ShiftEventType[] = ["start_of_day", "resume", "handover_in", "manager_takeover"];

export function deriveBoothState(
  latest: LatestEvent, wibDayStartMs: number,
): { state: BoothState; staffId: Id<"staff"> | null; staleAutoclose: boolean } {
  if (!latest || latest.type === "signoff_close") {
    return { state: "closed", staffId: null, staleAutoclose: false };
  }
  // A non-closed event from a prior WIB day = forgot to close → treat as closed.
  if (latest.created_at < wibDayStartMs) {
    return { state: "closed", staffId: null, staleAutoclose: true };
  }
  if (latest.type === "lock") return { state: "locked", staffId: latest.staff_id, staleAutoclose: false };
  if (latest.type === "handover_out") return { state: "handover_pending", staffId: latest.staff_id, staleAutoclose: false };
  if (OPEN_TYPES.includes(latest.type)) return { state: "open", staffId: latest.staff_id, staleAutoclose: false };
  return { state: "closed", staffId: null, staleAutoclose: false };
}

export function computeShiftHoursMs(shiftStartedAt: number, endedAt: number): number {
  return Math.max(0, endedAt - shiftStartedAt);
}
```

```typescript
import { v } from "convex/values";
import { query } from "../_generated/server";
import { internal } from "../_generated/api";
import { deriveBoothState } from "./lib";
import { wibDayWindow } from "../lib/time";

  args: { deviceId: v.string() },

    return { ...derived, staffName };
  },
});
```
