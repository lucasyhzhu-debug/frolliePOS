# FieldMessage — inline messaging over toasts (v1.2 #12 slice 1) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace synchronous client-side form-validation toasts with an inline, WCAG-AA `FieldMessage` design-system primitive in the two worst offenders (`mgr/products.tsx`, `mgr/vouchers.tsx`), fenced by an ESLint guard so they can't regress.

**Architecture:** New `cva` primitive (`fieldMessageVariants`, parallel to `badgeVariants`) on lifted-for-dark `error`/`success` semantic tokens; per-field error-object state in each form replaces `toast.error("literal")`; dynamic server errors and `toast.success` stay as toasts; a scoped `no-restricted-syntax` ESLint block bans literal-arg `toast.error`/`toast.warning` in the migrated files.

**Tech Stack:** React 19 + TypeScript + Vite · Tailwind CSS 4 (`@theme inline` token indirection) · shadcn/ui (new-york/stone) · `class-variance-authority` · `lucide-react` · Sonner (kept for async/global) · vitest + Testing Library · ESLint flat config + esquery.

## Global Constraints

- **Pure FE + lint + docs.** No backend, schema, Convex, or validation-LOGIC change — only the *delivery* of sync validation moves toast → inline. Validation conditions are byte-for-byte preserved.
- **Bucket discipline (ADR-048):** literal-string-first-arg `toast.error`/`toast.warning` = sync validation → **inline**. Dynamic-first-arg `toast.error(humanize*Error(err))` / `toast.error(msg)` = server/async → **stays toast**. `toast.success(...)` → **stays toast**.
- **Do NOT touch** login/PIN surfaces: `login.tsx`, `PinSheet`, `PinEntry`, `NumericKeypad` (owned by #11/#7). The existing `PinSheet error` prop is out of scope.
- **Tones shipped:** `error` + `success` only. No `warning`/`info` until a consumer needs them (ADR-047 no-dead-tokens ethos).
- **App is dark-only:** `<html class="dark">` (`index.html:2`). Token lift targets `.dark`; `:root` values kept for hygiene.
- **Token lift MUST live in `@theme inline`** (not a new static `@theme`) or `bg-error/15`-style opacity modifiers break.
- Commit message footer: `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.

```tsx
import * as React from "react";
import { cva, type VariantProps } from "class-variance-authority";
import { AlertCircle, CheckCircle2 } from "lucide-react";
import { cn } from "@/lib/utils";

// Inline form-validation message. Sanctioned primitive for sync validation
// (ADR-048) — anchored under the field, AA-legible on the phthalo canvas.
// Tone color rides the dark-lifted --color-error/--color-success tokens.
const fieldMessageVariants = cva(
  "flex items-start gap-1.5 rounded-r-sm border-l-2 py-0.5 pl-2 text-sm leading-snug",
  {
    variants: {
      tone: {
        error: "border-error text-error bg-error/10",
        success: "border-success text-success bg-success/10",
      },
    },
    defaultVariants: { tone: "error" },
  },
);

const TONE_ICON = { error: AlertCircle, success: CheckCircle2 } as const;
// error interrupts (role=alert ⇒ assertive); success is polite (role=status).
const TONE_ROLE = { error: "alert", success: "status" } as const;

export interface FieldMessageProps
  extends React.HTMLAttributes<HTMLParagraphElement>,
    VariantProps<typeof fieldMessageVariants> {}

export function FieldMessage({
  className,
  tone,
  children,
  ...props
}: FieldMessageProps) {
  const t = tone ?? "error";
  const Icon = TONE_ICON[t];
  return (
    <p
      role={TONE_ROLE[t]}
      className={cn(fieldMessageVariants({ tone }), className)}
      {...props}
    >
      <Icon className="mt-0.5 size-3.5 shrink-0" aria-hidden="true" />
      <span>{children}</span>
    </p>
  );
}

export { fieldMessageVariants };
```

## Conversion recipe (applies to Tasks 3 & 4 — read once)

The mechanical transform for every sync-validation site is identical:

**1. Add error-state scaffolding** near the top of the `*Inner` component:

```tsx
import { FieldMessage } from "@/components/ui/field-message";
// ...
const [errors, setErrors] = useState<Record<string, string>>({});
// Clear a single field's error (used on input change).
const clearFieldError = (k: string) =>
  setErrors((e) => {
    if (!(k in e)) return e;
    const { [k]: _omit, ...rest } = e;
    return rest;
  });
// Clear a whole dialog's errors by key prefix (used on open/close); no arg = all.
const clearErrors = (prefix?: string) =>
  setErrors((e) =>
    prefix
      ? Object.fromEntries(Object.entries(e).filter(([k]) => !k.startsWith(prefix)))
      : {},
  );
```

**2. Rewrite each validation handler to collect-all (not early-return)** — build a local `next` map, set every invalid field, and only proceed when empty. Worked example in Task 3 Step 1. Field keys are **namespaced by dialog** (`add.*`, `sku.*`, `meta.*`, `price.*`, `comp.*` / for vouchers `add.*`, `meta.*`) so dialogs don't collide.

**3. Render `<FieldMessage>` under each input** when its error is set:

```tsx
{errors["add.price"] && (
  <FieldMessage id="add.price-error">{errors["add.price"]}</FieldMessage>
)}
```

**4. Wire a11y on each input:** `aria-invalid={!!errors["add.price"]}` and `aria-describedby={errors["add.price"] ? "add.price-error" : undefined}`.

**5. Clear-on-edit:** in each input's `onChange`, after the existing `set…`, call `clearFieldError("<key>")`.

**6. Clear-on-open:** in each `open*()`/dialog-reset function, call `clearErrors("<dialog-prefix>")` (or `clearErrors()` where the whole dialog is reset).

**7. Focus the first errored field on submit:** define a per-dialog `fieldKey → inputId` map (input ids do NOT equal field keys), then after `setErrors(next)` with errors present, focus the first errored field in handler order:

```tsx
const firstBad = ORDER.find((k) => next[k]);            // ORDER = field keys in display order
if (firstBad) {
  const el = document.getElementById(FOCUS_MAP[firstBad]);
  el?.focus();
  el?.scrollIntoView({ block: "nearest" });
}
```

**8. Leave untouched:** every `toast.success(...)`, every `toast.error(humanize*Error(err))` / `toast.error(msg)` (dynamic server errors), and the `PinSheet error` prop.
