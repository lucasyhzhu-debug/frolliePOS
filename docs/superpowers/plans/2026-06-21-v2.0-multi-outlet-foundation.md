# v2.0 Multi-outlet Foundation (data plane) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Thread `outlet_id` through every operational table with session-derived scoping, so Frollie's one deployment runs multiple physical outlets, landed via a zero-downtime optional→backfill→enforce migration.

**Architecture:** SILO model (ADR-051): the Convex deployment *is* the business; `outlet_id` is the sole data-plane scoping column, derived structurally from the authenticated session (never a client arg). Every operational scan index leads with `outlet_id`; a new `index-leads-with-outlet_id` ESLint fence makes "forgot to scope" a CI failure. Staff are business-level identities with outlet access via a `staff_outlet_access` join. Per-outlet singletons (`pos_settings`, `pos_recount_state`) and outlet-prefixed receipt numbers (`R-<code>-YYYY-NNNN`).

**Tech Stack:** Convex 1.31.7 (TypeScript), React 19 + Vite, custom ESLint flat-config rules (`tools/eslint-rules/`), `convex-test` + vitest (edge-runtime).

**Spec:** `docs/superpowers/specs/2026-06-21-multi-tenancy-foundation-design.md`
**ADR:** `docs/ADR/051-multi-outlet-tenancy-silo.md`
**Review:** `docs/reviews/staffreview-v2.0-multi-outlet-foundation-spec-2026-06-21.md`

## Global Constraints

- **`outlet_id` type is `v.id("outlets")`** (real FK) everywhere — rows, sessions, helper params (Decision OQ1).
- **`outlet_id` is server-derived from the session, NEVER a public mutation/query arg** (ADR-051 Decision D).
- **Migration ordering is load-bearing:** Step 1 (add optional + new indexes) → write `outlet_id` in handlers → Step 2 (backfill) → Step 3 (enforce required + drop old indexes). Never enforce before backfill; never drop an index a live reader still uses (deploy-skew-fatal).
- **Cross-module DB reads go through the owning module's `_internal` API** (ADR-034); enforced by `no-cross-module-db-access`.
- **Public mutations require `idempotencyKey` + `withIdempotency` + `authCheck`** (ADR-013); the handler re-calls `require*Session` so authCheck runs before the cache lookup — keep the dual call.
- **`convex/lib/**` must be V8-safe** (no `"use node"`).
- **All money integer rupiah** (ADR-015); **all `_at` via `Date.now()` server-side** (ADR-031).
- **Audit:** every state-changing mutation calls `logAudit`; `audit_log.action` is a free `v.string()` (no enum to edit). Add `outlet_id` to metadata where a session exists.
- **Tables that do NOT get `outlet_id`** (backfill skips, fence ignores): `staff`, `audit_log`, `pos_settlements`, `api_tokens`, `api_rate_buckets`, `api_request_log`, `pos_idempotency`, `pos_device_activation_attempts`, `telegram_log`, `telegramUpdates`, `telegramChats`.
- **Tests:** `npx vitest run <file>` per task; full gate `npm run typecheck && npm run lint && npx vitest run` before merge.

```ts
export async function requireSession(
  ctx: QueryCtx | MutationCtx,
  sessionId: Id<"staff_sessions">,
): Promise<{ staffId: Id<"staff">; deviceId: string; role: "staff" | "manager"; outlet_id: Id<"outlets"> | undefined }> {
  const s = await ctx.db.get(sessionId);
  if (!s || s.ended_at != null) throw new Error("NO_SESSION");
  const staff = await ctx.db.get(s.staff_id);
  if (!staff || !staff.active) throw new Error("NO_SESSION");
  // Migration-tolerant window (I4): unstamped old sessions fall back to the single default outlet.
  let outlet_id = (s as any).outlet_id as Id<"outlets"> | undefined;
  if (!outlet_id) {
    const def = await ctx.db.query("outlets").withIndex("by_active", (q) => q.eq("active", true)).first();
    outlet_id = def?._id;
  }
  return { staffId: s.staff_id, deviceId: s.device_id, role: staff.role, outlet_id };
}

export async function requireManagerSession(ctx, sessionId) {
  const r = await requireSession(ctx, sessionId);
  if (r.role !== "manager") throw new Error("MANAGER_ONLY");
  return { staffId: r.staffId, deviceId: r.deviceId, outlet_id: r.outlet_id };
}
```

export const _allocateReceiptNumber_internal = internalMutation({
  args: { outletId: v.id("outlets"), year: v.number() },
  handler: async (ctx, { outletId, year }) => {
    const code = await ctx.runQuery(internal.outlets.internal._requireOutletCode_internal, { outletId });
    const counter = await ctx.db.query("pos_receipt_counters").withIndex("by_outlet_year", (q) => q.eq("outlet_id", outletId).eq("year", year)).first();
    let n: number;
    if (counter) { n = counter.next_number; await ctx.db.patch(counter._id, { next_number: n + 1 }); }
    else { n = 1; await ctx.db.insert("pos_receipt_counters", { outlet_id: outletId, year, next_number: 2 }); }
    return `R-${code}-${year}-${String(n).padStart(4, "0")}`;
  },
});
