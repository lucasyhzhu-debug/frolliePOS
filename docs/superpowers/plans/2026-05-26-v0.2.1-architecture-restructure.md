# v0.2.1 — Architecture Restructure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Refactor the v0.2.0 flat `convex/` codebase into the module-shaped layout per [ADR-034](../../ADR/034-deep-modules-surface-apis.md), add stable string identifiers (`staffCode` / `productCode` / `componentCode`), wire up a module-boundary lint rule as a hard CI gate, and scaffold the external API surface — **without changing any business behavior**.

**Architecture:** Deep modules with explicit `public.ts` / `internal.ts` / `schema.ts` fragments per ADR-034. Each `convex/<module>/` owns a clear surface. Foundational modules (`auth`, `idempotency`, `audit`) are allow-listed for cross-module imports. External API surface scaffolded under `convex/api/v1/` but **no endpoints in v0.2.1** (deferred to v0.3 with first transaction endpoint).

**Tech Stack:** TypeScript, Convex 1.31.7, React 19 + Vite, Tailwind 4, Vitest + convex-test + jsdom + edge-runtime, ESLint flat config (NEW — no config exists today), argon2id (auth carryover).

**Spec source:** [`docs/v0.2.1-restructure-scope.md`](../../v0.2.1-restructure-scope.md) — read this first for the ground-truth inventory + the 11 deferred items.

**Total tasks:** 32, organised into 10 sections (A–J). Module migrations are sequenced foundational-first to keep `npm test` green throughout.

# Section A: Setup & guardrails

This section adds the module-boundary lint rule and ESLint setup BEFORE any module migration. Critical ordering — if the rule lands after the migration, mid-migration violations are invisible.

> **Pattern (applies through Section F): after any cross-module move, update import paths in every still-existing consumer BEFORE running tests.** A module's import path changes the moment it moves (`./idempotency` → `./idempotency/internal`). Existing flat files that still import the old path will fail to resolve, breaking the entire test suite. Each task in Sections B–E that moves a module includes an explicit "update consumer imports" step. Grep pattern: `from "\./<moduleName>"` (or `"\.\./<moduleName>"` from one level deeper).

- [ ] **Step 3: Implement the rule**

Create `tools/eslint-rules/no-cross-module-db-access.js`:

```js
/**
 * ESLint rule: no-cross-module-db-access
 *
 * Per ADR-034, modules in convex/<module>/ must not access tables owned by
 * other modules directly. Cross-module access goes through the owner's
 * public.ts / internal.ts exports.
 *
 * Allow-list: foundational modules (auth, idempotency, audit) and seed are
 * exempt — they're either called by everyone or inherently cross-cutting.
 *
 * Top-level convex/*.ts files (pre-restructure) are treated as the "root"
 * module and excluded from checks (the restructure migrates them out).
 */
export default {
  meta: {
    type: "problem",
    docs: {
      description: "Disallow cross-module ctx.db.* access in convex/ modules per ADR-034",
    },
    schema: [{
      type: "object",
      properties: {
        ownership: {
          type: "object",
          additionalProperties: { type: "string" },
        },
        allowlist: {
          type: "array",
          items: { type: "string" },
          default: ["auth", "idempotency", "audit", "seed"],
        },
      },
      additionalProperties: false,
    }],
    messages: {
      crossModule:
        "Table '{{ table }}' is owned by module '{{ owner }}'; module '{{ caller }}' must use {{ owner }}/public.ts or {{ owner }}/internal.ts instead of direct ctx.db access.",
    },
  },
  create(context) {
    const opts = context.options[0] ?? {};
    const ownership = opts.ownership ?? {};
    const allowlist = new Set(opts.allowlist ?? ["auth", "idempotency", "audit", "seed"]);

    // Derive caller module from filename: convex/<module>/... or convex/<file>.ts (root).
    // Robust against Windows backslashes AND against repo paths that may contain
    // "convex" in a parent directory name. Anchors on the LAST occurrence of
    // "/convex/" or "^convex/" — the project's own convex directory.
    const filename = context.filename.replace(/\\/g, "/");
    const match =
      filename.match(/\/convex\/([^/]+?)(?:\/|\.ts$)/) ??
      filename.match(/^convex\/([^/]+?)(?:\/|\.ts$)/);
    if (!match) return {}; // not a convex file
    const segment = match[1];
    const isRoot = segment.endsWith(".ts") || filename.match(/\/convex\/[^/]+\.ts$/) || filename.match(/^convex\/[^/]+\.ts$/);
    const callerModule = isRoot ? "__root__" : segment;

    // Root files (pre-restructure) are exempt; allow-listed modules are exempt.
    if (callerModule === "__root__" || allowlist.has(callerModule)) {
      return {};
    }

    return {
      CallExpression(node) {
        // Match ctx.db.<method>("table_name", ...)
        const callee = node.callee;
        if (
          callee.type !== "MemberExpression" ||
          callee.object.type !== "MemberExpression" ||
          callee.object.object.type !== "Identifier" ||
          callee.object.object.name !== "ctx" ||
          callee.object.property.type !== "Identifier" ||
          callee.object.property.name !== "db"
        ) return;

        const firstArg = node.arguments[0];
        if (!firstArg || firstArg.type !== "Literal" || typeof firstArg.value !== "string") return;

        const table = firstArg.value;
        const owner = ownership[table];
        if (!owner || owner === callerModule) return;

        context.report({
          node,
          messageId: "crossModule",
          data: { table, owner, caller: callerModule },
        });
      },
    };
  },
};
```

import { QueryCtx, MutationCtx } from "../_generated/server";
import { Id } from "../_generated/dataModel";

/**
 * Resolve a session id to its staff context. Throws NO_SESSION if the session
 * is ended, missing, or its staff record is inactive.
 *
 * Extracted from convex/staff.ts in v0.2.1 to break the audit→staff backwards
 * dependency (ADR-034 §"Layer 1 — module boundaries"). Now lives in auth/
 * (foundational module) so audit/, staff/, and any future module can import it
 * without reaching across the staff/ boundary.
 */
export async function requireSession(
  ctx: QueryCtx | MutationCtx,
  sessionId: Id<"staff_sessions">,
): Promise<{ staffId: Id<"staff">; deviceId: string; role: "staff" | "manager" }> {
  const s = await ctx.db.get(sessionId);
  if (!s || s.ended_at != null) throw new Error("NO_SESSION");
  const staff = await ctx.db.get(s.staff_id);
  if (!staff || !staff.active) throw new Error("NO_SESSION");
  return { staffId: s.staff_id, deviceId: s.device_id, role: staff.role };
}

export async function requireManagerSession(
  ctx: QueryCtx | MutationCtx,
  sessionId: Id<"staff_sessions">,
): Promise<{ staffId: Id<"staff">; deviceId: string }> {
  const { staffId, deviceId, role } = await requireSession(ctx, sessionId);
  if (role !== "manager") throw new Error("MANAGER_ONLY");
  return { staffId, deviceId };
}

/**
 * Single payload for the catalog screen + offline cache. Persisted to IDB
 * client-side by useCatalogCache so cold starts work offline.
 * available_qty per product is computed client-side (ADR-017) from the
 * components + stockLevels in this payload.
 *
 * vouchers not in v0.2 — added in v0.6.

export const catalog = query({
  args: {},
  handler: async (ctx) => {
    const [products, skus, allComponents, allStockLevels] = await Promise.all([
      ctx.db
        .query("pos_products")
        .withIndex("by_active_sort", (q) => q.eq("active", true))
        .collect(),

        .query("pos_inventory_skus")

## Task F6: ~~Flip the 3 code fields to required~~ — DEFERRED to v0.3

**DO NOT EXECUTE THIS TASK in v0.2.1.** Deferred per staffreview Critical #2.

**Why deferred:** flipping `code` from optional to required cascade-breaks at least 3 hidden insert sites that the original plan missed:

1. **`_seedStaffCommit_internal`** (in `convex/auth/internal.ts` after Task C3) — test-only helper called by `seedStaff()` in every test file. Inserts `staff` without a `code` field.

3. **Raw `ctx.db.insert("staff", ...)` calls** in `convex/auth/__tests__/auth.test.ts` and likely other test files — explicit test inserts that don't supply `code`.

Flipping to required without updating all three breaks ~half the test suite AND the production createStaff mutation. Adding code-allocation logic to all three is its own design question (sequential `S-NNNN` based on `max(existing)+1` is the obvious choice but has race-safety implications under concurrent createStaff calls — needs explicit handling).

**v0.2.1 ships `code` as optional.** Seed populates codes for the 5 standard staff + 5 components + 7 products (Task F3). Format conformance tests (Task F5) verify the seed paths produce valid codes. Optional doesn't weaken the v0.2.1 contract because there's no external API consumer yet.

**v0.3 makes required** once:
1. `createStaff` mutation gains `S-NNNN` allocation logic (sequential, with idempotency-key-protected race handling).
2. (Future) `createProduct` / `createComponent` mutations gain similar allocation for `productCode` / `componentCode`.
3. Every test that inserts via `ctx.db.insert` directly is updated to supply codes.

When v0.3 lands the flip, schema migration ordering:
- a. Verify all existing rows have non-null codes (one-off migration query / dashboard inspection)
- b. Flip optional → required in schema
- c. Deploy — Convex validates on push, fails fast if any row missing

Replace its body with:

```markdown
### 1. Audit logging from any state-changing mutation

Every state-changing mutation calls `logAudit(ctx, {...})` from `convex/audit/internal.ts`. Mechanics:

- **`logAudit` is a plain async TypeScript function**, NOT an `internalMutation`. Called inside a `mutation` or `internalMutation`, it runs in the caller's transaction trivially (Convex mutation semantics) — atomicity is preserved without the JSON-serialization overhead of `ctx.runMutation` boundary crossings.
- The function is exported from `convex/audit/internal.ts` and imported by other modules. The import IS the API boundary; the foundational-module allow-list (`auth`, `idempotency`, `audit`) means cross-module imports of these helpers are sanctioned.
- Direct `ctx.db.insert("audit_log", ...)` from outside `convex/audit/internal.ts` is a CI lint block (custom ESLint rule).
- If a future need arises to invoke audit from an `internalAction` or cross-mutation orchestration, a thin `internalMutation` wrapper around `logAudit` can be added at that time. Don't pre-build it.
- The append-only guarantee from [ADR-007](./007-audit-log-append-only.md) is preserved: a thrown error in the caller mutation rolls back the audit row alongside the state change.

(Amended 2026-05-26 from v0.2.1 ground-truth walk per docs/v0.2.1-restructure-scope.md §3.5. The previous text prescribed `internalMutation`; in practice the plain-helper pattern is both atomic and cheaper.)
```
