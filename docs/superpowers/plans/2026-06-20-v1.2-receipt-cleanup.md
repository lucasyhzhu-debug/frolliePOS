# v1.2 #13 Receipt Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Clean up the printed/digital receipt — drop the redundant paid badge, collapse the payment block to one line, fix the manual-BCA method leak, and move the footer default to English — without touching the receipt body's Indonesian copy.

**Architecture:** One `ReceiptViewModel` contract feeds two independent renderers (`template.ts` = HTML/server, `escpos.ts` = thermal/client). Both stay method-agnostic; the only method-aware change is a single upstream label derivation in `receipts/internal.ts` that branches on the txn's `confirmed_via`. Renderer shape is tested with hand-built VMs (fast, no DB); label derivation is tested via `convex-test` through `getReceiptForPrint`.

**Tech Stack:** Convex 1.31.7, TypeScript, Vitest + convex-test, esc-pos-encoder.

## Global Constraints

- **Money as integer rupiah**, formatted via `src/lib/format.ts` — untouched here (business rule #14).
- **i18n boundary (ADR-049):** receipts are out of scope for v1.2 i18n. The receipt **body stays Indonesian** (`Subtotal`, `Pengembalian`, `Simpan struk ini…`, line annotations). Only the configurable footer default moves to English.
- **Method-agnostic renderers:** neither renderer may branch on `payment_method` string values. Method labelling lives only in `receipts/internal.ts::buildVmFromTxnWithLines`.
- **No schema change** — `pos_settings` footer/business-name fields and `pos_transactions.confirmed_via` already exist.
- **Thermal is ASCII-only:** `escpos.ts::ascii()` strips any char outside `\x20-\x7E`. The HTML payment separator is a middot `·`; the **thermal separator must be an ASCII hyphen `-`** (a middot would be silently folded away). Deliberate per-medium divergence.
- **Suppress the paid badge ONLY for `status === "paid"`** — refund-state badges (`SEBAGIAN DIKEMBALIKAN`, `DIKEMBALIKAN`) are kept verbatim.
- **`getReceiptForPrint`'s `statusLabel` projection is unchanged** — it still returns the label string; the *thermal renderer* decides whether to print it. Do not alter that query.
