## Per-task detail

### T1 — `kind` as a pure annotation (`xendit.ts`)
TDD: first update the **6** existing `.toEqual` call sites (lines 59,70,81,90,97,98 — line 98 shares an `it` with 97, so 6 call sites across 5 `it` blocks) to include `kind`, add the C1 test (refund envelope → `paid`/`matchKey` identical to today) and the classification cases (RED), then implement.
- Extend `WebhookParse` with `kind: WebhookKind` (`"qr_payment" | "bca_va" | "refund" | "ignored"`).
- In `parseXenditWebhook`, keep every existing branch's `paid`/`matchKey`/`amount`/`receiptId`/`paymentSource` **byte-identical**. Derive `kind` alongside: refund if `event`/`data.type`/`p.type` contains `"refund"`; `bca_va` if the FVA branch matched; `qr_payment` if the SUCCEEDED-QR branch matched; else `ignored`. **`kind` must never feed back into `paid`/`matchKey`** (staffreview C1; honors `xendit.ts:150-156`).
- Verify: `npm run test:convex -- xendit`.

### T2 — outbox table (`schema.ts`)
- Add `pos_qris_forward_outbox` (fields per spec §4.2) with `.index("by_xendit_qr_id", ["xendit_qr_id"])` and `.index("by_status_next", ["status","next_attempt_at"])`. No root-schema edit (`...paymentsTables` auto-wires). Run `npx convex codegen`.

### T3 — forwarder (`forwarder.ts`) — DEFAULT runtime, no `"use node"`
- `_enqueueForward_internal({ raw_payload, xendit_qr_id })` (internalMutation): dedup via `by_xendit_qr_id` (existing row → return); insert one `pending` row; `ctx.scheduler.runAfter(0, internal.payments.forwarder._deliverForward, { id })`.
- `_deliverForward({ id })` (internalAction): read row; not `pending` → return. `fetch(RM_QR_WEBHOOK, { method:"POST", headers:{ "content-type":"application/json", "x-callback-token": env, "x-frollie-forward-secret": env }, body: raw_payload })`. `res.ok` → `delivered`+`delivered_at`. 401 → `failed` + ops report (terminal). Other non-ok/throw → `attempts++`, backoff `next_attempt_at`, reschedule; `>= MAX_ATTEMPTS` → `failed` + ops report. `const RM_QR_WEBHOOK = "https://decisive-wombat-7.convex.site/api/xendit/qr-payment"` (hardcoded, SSRF-safe). Retry on ANY non-terminal failure (NOT `cronRetry.isTransientError`). Ops: `_recordError_internal({ kind:"backend", route:"convex/payments/forwarder", message })`.
- Tests (mock `fetch`): 200→delivered; 500→attempts++/pending/rescheduled; MAX→failed+report; 401→failed+report; enqueue dedup (2nd same qr_id → 1 row).

### T4 — wire into `xenditWebhook` (`webhook.ts`)
- Add `kind` to the `parseXenditWebhook` destructure (line ~52). After the existing paid processing (after line ~69, inside the try), if `kind === "qr_payment" && matchKey && process.env.FROLLIE_FORWARD_ENABLED === "true"` → `await ctx.runMutation(internal.payments.forwarder._enqueueForward_internal, { raw_payload: raw, xendit_qr_id: matchKey })` wrapped so a throw never breaks the 200 (mirror existing `reportWebhookError`). POS's own path (lines 54-69) UNCHANGED.
- Tests: `qr_payment` body enqueues (kill-switch on); `refund`/`bca_va`/`ignored`/kill-switch-off → no enqueue; existing 401/dup/cancelled/mismatch assertions still pass; `_onPaidWebhook_internal` still invoked as before.

### T5 — verification
- `npm run typecheck` (`tsc -b && tsc -p convex`) + `npm run test:convex` green. Audit: paid path untouched, no secret in outbox rows, 401 terminal, hardcoded target, kill-switch present.
