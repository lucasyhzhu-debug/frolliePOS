# Phthalo-dark Design System Implementation Plan (v1.2 Phase 1 — #2 + #4 + #5)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Port the Frollie/Lucas phthalo-dark canvas into the POS as the default theme, enrich the shadcn primitives + 3 highest-leverage surfaces with real hierarchy and engaging motion, and fold in the staff-home declutter (#4) and top-left lock icon (#5).

**Architecture:** Dark-by-default via a permanent `.dark` class on `<html>`; phthalo palette lives in `.dark`, `:root` retained as the enriched-light glare-gate fallback (flip one class to fall back). Tokens drive everything, so the lift propagates to untouched routes. Motion = `tw-animate-css` for Radix primitives + Framer Motion micro-interactions. A throwaway `design-mock.html` is approved by Lucas before app-wide propagation (hard STOP).

**Tech Stack:** React 19 + TypeScript + Vite, Tailwind CSS 4 (CSS-config in `src/index.css`), shadcn/ui (new-york/stone), Framer Motion ^12 (already in deps), tw-animate-css (new dev dep), lucide-react.

## Global Constraints

- **Direction is phthalo-DARK by default** (roadmap-locked). Colors from `frollie-pos design files/lucas-frollie-design-system/project/colors_and_type.css`: paper `#102821`, card `#163630`, elevated `#1E4740`, warm ink `#F1E9D8`, teal primary, citrus `#F9A84A` accent.
- **HARD GATE (locked):** real-booth-tablet readability/glare check under mall lighting before declaring rollout done. Dev pass via chrome-devtools-mcp emulated tablet viewport. Fallback if it washes out = remove `class="dark"` (the enriched-light `:root`).
- **Design sign-off gate:** Lucas approves `design-mock.html` screenshots before propagating tokens to live surfaces (Task 1 STOP).
- **Do NOT touch** `src/lib/format.ts` `id-ID` currency formatting (currency, not copy — i18n is #1).
- **Do NOT implement** `PinEntry` submit/spinner/inline-error logic or the keypad press/pending *interaction* (owned by #7/#11). #2 = visual/token restyle of login + keypad colors only.
- **Do NOT remove** any payment-method schema literal or backend code (no backend changes in this phase at all).
- Honor `prefers-reduced-motion`: every Framer Motion interaction no-ops under it (use `useReducedMotion`).
- Single squash-merge PR (repo convention). `docs/CHANGELOG.md` entry required.
- Preserve all existing test hooks: `data-testid`s, `aria-label`s, and route `href`s through every redesign.
