# v0.5.3c — Settlements Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ingest Xendit settlement (payout-to-merchant) data via a nightly poll-cron + a manager manual-entry fallback, surfaced as a read-only `/settlements` view with a per-day reconciliation cross-check.

**Architecture:** New deep `convex/settlements/` module (ADR-034). Xendit protocol stays in the existing `payments/xendit.ts` adapter (new `fetchSettledTransactions`). A `"use node"` resilient cron action polls `GET /transactions` for settled rows, aggregates by `settlement_date`, and upserts `pos_settlements`. A manager-session mutation (`recordSettlement`) is the manual fallback. Reconciliation matches Xendit `reference_id` (`pos-${txnId}`) back to `pos_transactions`. **No money ever moves** — read-only `GET` + recording.

**Tech Stack:** Convex 1.31.7 (internalAction `"use node"`, internalMutation, query/mutation, crons), React 19 + Vite, vitest + convex-test (edge-runtime), Tailwind 4 + shadcn.

**Source spec:** `docs/superpowers/specs/2026-06-02-v0.5.3c-settlements-design.md` · **Staffreview:** `docs/reviews/staffreview-v0.5.3c-settlements-spec-2026-06-02.md` · **API facts:** `docs/xendit-reference/settlement-reconciliation.md`

**Branch:** implement on a fresh worktree cut from `origin/main` (the planning artifacts land on `main` first). Commit per task: `feat(settlements): …`.

## Resolved design decisions (from spec Open Items)

- **`bca_account_destination`:** manual-entry may set it; **cron leaves it `undefined`**. No new `pos_settings` field in v0.5.3c (keeps the slice tight). View renders `—` when absent.
- **`xendit_settlement_id`:** synthesised `xnd-settle-${settlement_date}` (no batch-id object exists).
- **`settlement_date`:** ISO `YYYY-MM-DD` in WIB — the single dedup key.
- **Settled statuses counted:** both `SETTLED` and `EARLY_SETTLED`.
- **Manual vs auto collision on same date:** a cron `xendit_sync` upsert **supersedes** a `manual` row for that date (the manual entry's audit row preserves the history). Asserted in a test.
- **`syncSettlementsNow`:** **included** (manager-gated on-demand re-poll), auth via `getSession`+role.
- **`payload`:** JSON string of the contributing-txn snapshot `[{reference_id, amount, fee}]`; absent for manual rows.

```typescript
// Pure settlement logic — V8-safe (NO "use node", no Convex imports). Unit-tested
// without a harness. Money is integer rupiah throughout (rule #14, ADR-015).
//
// The aggregator takes a STRUCTURAL minimum (referenceId/settlementDate/amount/fee)
// so the adapter's richer `SettledTxn` (payments/xendit.ts — the canonical
// normalized wire row) is assignable without a duplicate type name across modules.

export type DayAgg = {
  settlementDate: string;
  gross: number;
  mdr: number;
  net: number;
  count: number;
  refs: { reference_id: string; amount: number; fee: number }[];
};

export type PaidTxnRef = {
  txnId: string;
  receiptNumber?: string;
  total: number;
  paidAt: number;
};

export type ReconResult = {
  settled: PaidTxnRef[];
  pending: PaidTxnRef[];
  unmatched: string[]; // settled Xendit refs with no matching paid pos_transaction
};

/** Xendit reference_id is `pos-${txnId}` (retry: `pos-${txnId}-r-${uuid}`). */
export function parseRefToTxnId(ref: string): string {
  return ref.replace(/^pos-/, "").split("-r-")[0];
}

/** Group settled txns by settlement_date → integer-rupiah aggregate + snapshot.
 *  Structural param so the adapter's `SettledTxn` (a superset) is assignable. */
export function aggregateBySettlementDate(
  txns: { referenceId: string; settlementDate: string; amount: number; fee: number }[],
): DayAgg[] {
  const byDate = new Map<string, DayAgg>();
  for (const t of txns) {
    // Guard integer rupiah (IDR has no cents); a stray decimal must not slip through.
    const amount = Math.round(t.amount);
    const fee = Math.round(t.fee);
    let agg = byDate.get(t.settlementDate);
    if (!agg) {
      agg = {
        settlementDate: t.settlementDate,
        gross: 0,
        mdr: 0,
        net: 0,
        count: 0,
        refs: [],
      };
      byDate.set(t.settlementDate, agg);
    }
    agg.gross += amount;
    agg.mdr += fee;
    agg.count += 1;
    agg.refs.push({ reference_id: t.referenceId, amount, fee });
  }
  for (const agg of byDate.values()) agg.net = agg.gross - agg.mdr;
  return [...byDate.values()];
}

/** Classify paid txns vs the set of settled Xendit refs. */
export function reconcileDay(settledRefs: string[], paidTxns: PaidTxnRef[]): ReconResult {
  const settledTxnIds = new Set(settledRefs.map(parseRefToTxnId));
  const settled: PaidTxnRef[] = [];
  const pending: PaidTxnRef[] = [];
  for (const t of paidTxns) {
    if (settledTxnIds.has(t.txnId)) settled.push(t);
    else pending.push(t);
  }
  const paidIds = new Set(paidTxns.map((t) => t.txnId));
  const unmatched = settledRefs.filter((r) => !paidIds.has(parseRefToTxnId(r)));
  return { settled, pending, unmatched };
}
```

```typescript
// ─── Settlement reconciliation (v0.5.3c) ─────────────────────────────────────
// There is NO settlement webhook (docs/xendit-reference/settlement-reconciliation.md).
// Settlement is read by polling GET /transactions for settled rows.

export type SettledTxn = {
  referenceId: string;
  settlementStatus: string;
  settlementDate: string; // ISO YYYY-MM-DD
  amount: number;
  fee: number;
  channelCode?: string;
  currency?: string;
  createdAt?: string;
};

const SETTLEMENT_PAGE_LIMIT = 100;

/** Pure query-string builder. Exported so a test can assert the settled filter
 *  and date bounds without a live call (mirrors buildQrisHeaders). */
export function buildTransactionsQuery(window: { fromMs: number; toMs: number }): string {
  const iso = (ms: number) => new Date(ms).toISOString().slice(0, 19); // drop millis/Z noise
  const p = new URLSearchParams();
  // Both settled statuses are money-in-balance (SETTLED + EARLY_SETTLED).
  p.append("settlement_status", "SETTLED");
  p.append("settlement_status", "EARLY_SETTLED");
  p.set("created[gte]", iso(window.fromMs));
  p.set("created[lte]", iso(window.toMs));
  p.set("limit", String(SETTLEMENT_PAGE_LIMIT));
  return p.toString();
}

/** Pure normaliser — the single place raw Xendit field names are read. */
export function normalizeSettledTxn(raw: any): SettledTxn {
  return {
    referenceId: raw.reference_id,
    settlementStatus: raw.settlement_status,
    settlementDate: raw.settlement_date,
    amount: raw.amount,
    fee: raw.fee,
    channelCode: raw.channel_code,
    currency: raw.currency,
    createdAt: raw.created,
  };
}

/** Poll GET /transactions for settled rows over [fromMs, toMs], following
 *  pagination to completion. Read-only; never moves money. Buffer auth → the
 *  CALLER must run under "use node". */
export async function fetchSettledTransactions(window: {
  fromMs: number;
  toMs: number;
}): Promise<SettledTxn[]> {
  const out: SettledTxn[] = [];
  let afterId: string | undefined;
  // Bounded loop: hard cap pages to avoid an unbounded fetch on a bad cursor.
  for (let page = 0; page < 1000; page++) {
    const qs = buildTransactionsQuery(window) + (afterId ? `&after_id=${afterId}` : "");
    const res = await fetch(`${XENDIT_BASE}/transactions?${qs}`, {
      method: "GET",
      headers: { Authorization: authHeader(), "Content-Type": "application/json" },
    });
    if (!res.ok) {
      throw new Error(`XENDIT_TXNS_FAILED: ${res.status} ${await res.text()}`);
    }
    const json = (await res.json()) as { data?: any[]; has_more?: boolean };
    const rows = json.data ?? [];
    for (const r of rows) out.push(normalizeSettledTxn(r));
    if (!json.has_more || rows.length === 0) break;
    afterId = rows[rows.length - 1]?.id;
    if (!afterId) break;
  }
  return out;
}
```
> Note: `authHeader` and `XENDIT_BASE` already exist at the top of `xendit.ts` (lines 13, 34). The retry/pagination param names (`after_id`, `has_more`, `created[gte]`) are the asserted-not-verified bits to confirm in Step 0.

import { v } from "convex/values";
import { internalMutation } from "../_generated/server";

// Single writer for both ingress paths (cron + manual). Upsert by settlement_date.
// A cron (xendit_sync) upsert supersedes a manual row for the same date; the
// manual entry's audit row preserves the history (the audit is written by the
// public mutation, not here).
export const _upsertSettlement_internal = internalMutation({
  args: {
    settlementDate: v.string(),
    gross: v.number(),
    mdr: v.number(),
    net: v.number(),
    count: v.number(),
    source: v.union(v.literal("xendit_sync"), v.literal("manual")),
    bankLast4: v.optional(v.string()),
    payload: v.optional(v.string()),
    actorId: v.optional(v.id("staff")),
  },
  handler: async (ctx, args) => {
    const now = Date.now();
    const existing = await ctx.db
      .query("pos_settlements")
      .withIndex("by_settlement_date", (q) => q.eq("settlement_date", args.settlementDate))
      .first();

    const fields = {
      xendit_settlement_id: `xnd-settle-${args.settlementDate}`,
      settlement_date: args.settlementDate,
      gross_amount: args.gross,
      mdr_amount: args.mdr,
      net_amount: args.net,
      transaction_count: args.count,
      source: args.source,
      updated_at: now,
      ...(args.bankLast4 !== undefined ? { bca_account_destination: args.bankLast4 } : {}),
      ...(args.payload !== undefined ? { payload: args.payload } : {}),
    };

    if (existing) {
      await ctx.db.patch(existing._id, fields);
      return existing._id;
    }
    return await ctx.db.insert("pos_settlements", {
      ...fields,
      created_at: now,
      ...(args.actorId !== undefined ? { created_by: args.actorId } : {}),
    });
  },
});
```

import { internal, api } from "../_generated/api";
import { fetchSettledTransactions } from "../payments/xendit";
import { aggregateBySettlementDate } from "./lib";
import {
  isTransientError,
  resilientRetryDelayMs,
  RESILIENT_MAX_ATTEMPTS,
} from "../lib/cronRetry";

const DEFAULT_LOOKBACK_DAYS = 7;

// Core recon: poll settled txns over the lookback window, aggregate by
// settlement_date, upsert each. Idempotent (upsert-by-date) → re-run safe.
export const _reconcileSettlements = internalAction({
  args: { lookbackDays: v.optional(v.number()) },
  handler: async (ctx, args): Promise<{ ok: true; datesUpserted: number; txnCount: number }> => {
    const days = args.lookbackDays ?? DEFAULT_LOOKBACK_DAYS;
    const toMs = Date.now();
    const fromMs = toMs - days * 86_400_000;

    const settled = await fetchSettledTransactions({ fromMs, toMs });
    const aggs = aggregateBySettlementDate(settled);
    for (const a of aggs) {
      await ctx.runMutation(internal.settlements.internal._upsertSettlement_internal, {
        settlementDate: a.settlementDate,
        gross: a.gross, mdr: a.mdr, net: a.net, count: a.count,
        source: "xendit_sync",
        payload: JSON.stringify(a.refs),
      });
    }
    return { ok: true, datesUpserted: aggs.length, txnCount: settled.length };
  },
});

// Cron entry-point — resilient wrapper mirroring foundersSummary (cronRetry).
export const sendSettlementReconResilient = internalAction({
  args: { attempt: v.optional(v.number()) },
  handler: async (ctx, args): Promise<{ ok: true } | { ok: true; retried: true; nextAttempt: number }> => {
    const attempt = args.attempt ?? 0;
    try {
      await ctx.runAction(internal.settlements.actions._reconcileSettlements, {});
      return { ok: true as const };
    } catch (err) {
      if (isTransientError(err) && attempt + 1 < RESILIENT_MAX_ATTEMPTS) {
        await ctx.scheduler.runAfter(
          resilientRetryDelayMs(attempt),
          internal.settlements.actions.sendSettlementReconResilient,
          { attempt: attempt + 1 },
        );
        return { ok: true as const, retried: true as const, nextAttempt: attempt + 1 };
      }
      throw err; // non-transient / exhausted → surfaces in the cron dashboard
    }
  },
});

// On-demand manager re-poll. Actions can't use requireManagerSession/withIdempotency;
// assert via getSession + role. Replay-safe via the idempotent upsert writer.
export const syncSettlementsNow = action({
  args: { sessionId: v.id("staff_sessions"), lookbackDays: v.optional(v.number()) },
  handler: async (ctx, args): Promise<{ ok: true; datesUpserted: number; txnCount: number }> => {
    const session = await ctx.runQuery(api.auth.public.getSession, { sessionId: args.sessionId });
    if (!session || session.staff.role !== "manager") throw new Error("MANAGER_ONLY");
    return await ctx.runAction(internal.settlements.actions._reconcileSettlements, {
      lookbackDays: args.lookbackDays,
    });
  },
});
```

```typescript
import { v } from "convex/values";
import { query } from "../_generated/server";
import { internal } from "../_generated/api";
import { requireSession } from "../auth/sessions";
import { reconcileDay, parseRefToTxnId, type PaidTxnRef } from "./lib";

const DEFAULT_LIST_DAYS = 30;

// ADR-012: readable by any authenticated session (staff + manager).
export const listSettlements = query({
  args: { sessionId: v.id("staff_sessions"), days: v.optional(v.number()) },
  handler: async (ctx, args) => {
    await requireSession(ctx, args.sessionId); // throws on invalid; no role gate
    const rows = await ctx.db
      .query("pos_settlements")
      .withIndex("by_settlement_date")
      .order("desc")
      .collect();
    const cutoff = args.days ?? DEFAULT_LIST_DAYS;
    return rows.slice(0, cutoff); // ≤1 row/day; bounded on principle (v0.5.3a lesson)
  },
});

// Reconciliation cross-check for one settlement_date row.
export const getSettlementDetail = query({
  args: { sessionId: v.id("staff_sessions"), settlementDate: v.string() },
  handler: async (ctx, args) => {
    await requireSession(ctx, args.sessionId);
    const row = await ctx.db
      .query("pos_settlements")
      .withIndex("by_settlement_date", (q) => q.eq("settlement_date", args.settlementDate))
      .first();
    if (!row) return null;

    const settledRefs: string[] = row.payload
      ? (JSON.parse(row.payload) as { reference_id: string }[]).map((r) => r.reference_id)
      : [];

    // Reuse the existing lightweight paid-txn reader (transactions-owned, ADR-034).
    // Lookback window = 8 days back from the settlement date end (covers T+1..T+7).
    const dayEndMs = Date.parse(`${args.settlementDate}T17:00:00Z`); // ~end of WIB day
    const sinceMs = dayEndMs - 8 * 86_400_000;
    const paidDocs = await ctx.runQuery(
      internal.transactions.internal._listPaidTxnsSince_internal,
      { sinceMs },
    );
    const paidTxns: PaidTxnRef[] = paidDocs.map((d) => ({
      txnId: String(d._id),
      receiptNumber: d.receipt_number,
      total: d.total,
      paidAt: d.paid_at ?? 0,
    }));

    return { row, reconciliation: reconcileDay(settledRefs, paidTxns) };
  },
});
```

Append the mutation:
```typescript
// Manual fallback. Manager-SESSION (not PIN): a "tally what already happened"
// bookkeeping write — same class as markRefundSettled (rule #21/#22). The cron
// writes the identical row with no auth, so a PIN here would be ceremony. Audited.
export const recordSettlement = mutation({
  args: {
    idempotencyKey: v.string(),
    sessionId: v.id("staff_sessions"),
    settlementDate: v.string(),
    gross: v.number(),
    mdr: v.number(),
    net: v.number(),
    count: v.number(),
    bankLast4: v.optional(v.string()),
  },
  handler: withIdempotency<
    {
      idempotencyKey: string;
      sessionId: Id<"staff_sessions">;
      settlementDate: string;
      gross: number; mdr: number; net: number; count: number;
      bankLast4?: string;
    },
    { ok: true; settlementId: Id<"pos_settlements"> }
  >(
    "settlements.recordSettlement",
    async (ctx, args) => {
      const { staffId, deviceId } = await requireManagerSession(ctx, args.sessionId);
      if (!/^\d{4}-\d{2}-\d{2}$/.test(args.settlementDate)) throw new Error("DATE_INVALID");
      if (!Number.isInteger(args.gross) || !Number.isInteger(args.mdr) || !Number.isInteger(args.net))
        throw new Error("AMOUNT_NOT_INTEGER");
      if (args.net !== args.gross - args.mdr) throw new Error("NET_MISMATCH");
      if (args.count < 0) throw new Error("COUNT_INVALID");

      const settlementId = await ctx.runMutation(
        internal.settlements.internal._upsertSettlement_internal,
        {
          settlementDate: args.settlementDate,
          gross: args.gross, mdr: args.mdr, net: args.net, count: args.count,
          source: "manual",
          bankLast4: args.bankLast4,
          actorId: staffId,
        },
      );

      await logAudit(ctx, {
        actor_id: staffId,
        action: "settlement.recorded",
        entity_type: "pos_settlements",
        entity_id: String(settlementId),
        source: "booth_inline",
        device_id: deviceId,
        metadata: { settlement_date: args.settlementDate, net: args.net },
      });
      return { ok: true as const, settlementId };
    },
    {
      authCheck: async (ctx, args) => {
        await requireManagerSession(ctx, args.sessionId);
      },
    },
  ),
});
```

If the executing agent hits a runtime restriction, inline the upsert logic by importing a plain helper instead; but the `ctx.runMutation`-to-internalMutation path is the established pattern (see `refunds/public.ts` calling `_commitRefund_internal`). Verify at impl.
