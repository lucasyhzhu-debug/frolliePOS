# v0.3 — Sale flow + Xendit + PIN management Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the operational core of the booth — staff take a sale end-to-end on the Android device (cart → QRIS/BCA VA → receipt), drafts save and resume, voucher apply, three-path payment confirmation (webhook + polling + manager-PIN override), reconciliation on reload, plus the PIN management trio (bootstrap, self-change, manager-reset) and the off-booth Telegram-driven PIN reset that doubles as the v0.4 approval pattern's first real use.

**Architecture:** Five new Convex modules per ADR-034 (`transactions/`, `payments/`, `inventory/`, `vouchers/`, `approvals/`) plus extensions to `auth/`, `catalog/`, `seed/`, `telegram/`, and `http.ts`. The single-funnel `transactions.internal._confirmPaid_internal` is THE convergence point for all three confirmation paths. Client-side Zustand cart commits once on Save/Charge — no per-line server mutations. Frontend hook drives polling cadence (2s × 30 = 60s ceiling). Paid-only stock decrement (`_recordSaleMovement_internal` only called from `_confirmPaid`). All five new modules respect the ADR-034 boundary lint (no cross-module DB access; `audit/internal.logAudit` and `idempotency/internal.withIdempotency` are the allow-list helpers). Action-level idempotency uses a two-stage pattern: `_lookup_internal` short-circuit at action start, then the committing internal mutation writes the cache row in the same transaction as state changes; Xendit-touching actions also pass `X-IDEMPOTENCY-KEY` to Xendit for upstream dedup.

**Tech Stack:** Convex 1.31.7 (existing) + new modules; Zustand 5 (existing); IDB (existing) for offline queue + idempotency persistence; vite-plugin-pwa (existing); hash-wasm argon2id (existing — reused for all PIN work); Sonner toasts (existing); shadcn primitives (existing — extend with `PinSheet`); Xendit Invoice API; Telegram bot token (existing POC) graduates to production-pattern via ADR-035.

**ADR coverage in this phase:** 008/010/011/013/014/015/018/023/025/026/029/031/032/034 (referenced) + new ADR-035 (Telegram as internal comms channel — supersedes ADR-027 + ADR-033). Strategic foundations §4 (PPN schema-ready), §8 (three-path payment confirmation).

**Out of scope for v0.3** (verbatim from spec §"Out of scope"): refunds (v0.5), receipt PDF/sharing/printer (v0.5), transaction history list (v0.5), stock-in/spoilage/manager stock tools (v0.5/v0.6), manager dashboard + manual-override surveillance UI (v0.5), other approval kinds beyond `staff_pin_reset` (v0.4), founders shift-summary auto-post (v0.4), `routes/mgr/home.tsx` (v0.4), prod Telegram bot (v1.0), prod Convex cutover (v1.0), E2E browser tests (v0.6), `product_master` cross-deployment integration (v1.1+), voucher manager CRUD UI (v0.5/v0.6).

- **PR scope:** schema + backend + frontend + docs in one PR. v0.2 set the precedent that phase-sized changes ship as a single reviewable unit.

## Rollback strategy

- **Per-task atomic commits.** Each task ends with one commit. `git revert <hash>` is safe — the previous task's tests remain green by construction.
- **Backout the whole phase:** `git revert --no-commit <first-task-hash>^..HEAD && git commit -m "revert: v0.3"`.
- **Schema rollback (dev):** edit `convex/schema.ts` back to the previous shape. The `pos_stock_levels` move (Task 2) is the only "move" — to roll back, restore the table definition in `convex/catalog/schema.ts` and remove it from `convex/inventory/schema.ts`. Data persists across the move (Convex tracks tables by name; the module split is purely TypeScript organization).
- **Vercel rollback:** redeploy the previous tag via the Vercel dashboard if a bad build reaches preview.
- **Telegram side effects:** if `notifyStaffLockout` accidentally fires during testing, no data corruption — the manager just gets a stray approval message. The 60-minute token TTL self-cleans.

The existing `withIdempotency` already writes the cache row in the same transaction as the handler — that's how Convex mutations work. The semantic gap is for ACTIONS calling internal mutations: the action's `_lookup_internal` check happens before the side-effecting HTTP call, then the action calls an internal mutation that commits state + cache atomically. What we actually need is a small addition: explicit doc + 2 new test-only mutations. Update the doc comment in `withIdempotency`:

```ts
/**
 * Wrap a mutation handler so the same idempotencyKey replays the stored response
 * instead of re-executing. Public mutations MUST go through this. See ADR-013.
 *
 * Errors are NOT cached — a handler that throws does not write to pos_idempotency
 * (because the entire mutation transaction rolls back), so a retry with the same
 * key re-executes. v0.3 confirms this is the correct semantics for both mutation
 * callers (public.ts) and ACTION callers (actions.ts that runMutation an internal
 * commit). The Convex transactional guarantee means the cache row and the state
 * change are atomic: if the handler succeeds, both are committed; if it throws,
 * both are rolled back.
 *
 * ACTION-LEVEL IDEMPOTENCY PATTERN (v0.3+):
 * Actions that perform side-effecting HTTP (Xendit, Telegram) MUST:
 *   1. Pre-check the cache via internal.idempotency._lookup_internal(key).
 *      If hit → return parsed cached response (skip everything).
 *   2. Perform the HTTP call with X-IDEMPOTENCY-KEY = args.idempotencyKey
 *      where the upstream API supports it (Xendit Invoice API does).
 *   3. Call a wrapped internal mutation that commits Convex state. The
 *      withIdempotency wrap on THAT mutation writes the cache row atomically
 *      with the state change.
 *
 * This guarantees: first call's HTTP + state + cache all succeed atomically OR
 * none commit. A retry that re-fires HTTP gets Xendit's dedup + a fresh
 * mutation that's idempotent at the Convex layer.
 *
 * If `staffIdFromArgs` is provided, the resolved staff_id is stored on the
 * idempotency row for debugging. Pre-auth mutations (e.g. activateDevice)
 * omit it.
 */
```

The existing implementation does NOT need code changes; what we add is two new test-only mutations that exercise the action-style pattern:

/**
 * Record stock decrements for a paid transaction. Called ONLY from
 * transactions.internal._confirmPaid_internal — paid-only decrement per
 * strategic foundations §8 + ADR-018.
 *
 * Idempotent: each (source_transaction_line_id, inventory_sku_id) pair is a
 * unique-constraint emulation via the by_line_and_sku index. ADR-026 — guards
 * against webhook + polling race writing duplicate movements.
 *
 * Also updates pos_stock_levels.on_hand (may go negative — ADR-018) and
 * pos_stock_levels.updated_at (mandatory per staffreview Improvement #4).
 */
export const _recordSaleMovement_internal = internalMutation({
  args: {
    transactionId: v.id("pos_transactions"),
    lines: v.array(v.object({
      lineId: v.id("pos_transaction_lines"),
      skuId: v.id("pos_inventory_skus"),
      qty: v.number(), // positive — function negates internally
    })),
  },
  handler: async (ctx, args) => {
    const now = Date.now();
    for (const line of args.lines) {
      // ADR-026 dedup: skip if movement already exists for this line+sku pair
      const existing = await ctx.db
        .query("pos_stock_movements")
        .withIndex("by_line_and_sku", (q) =>
          q.eq("source_transaction_line_id", line.lineId).eq("inventory_sku_id", line.skuId),
        )
        .first();
      if (existing) continue;

      await ctx.db.insert("pos_stock_movements", {
        inventory_sku_id: line.skuId,
        qty: -line.qty,
        source: "sale",
        source_transaction_line_id: line.lineId,
        created_at: now,
      });

      // Decrement the on_hand cache (may go negative per ADR-018)
      const level = await ctx.db
        .query("pos_stock_levels")
        .withIndex("by_sku", (q) => q.eq("inventory_sku_id", line.skuId))
        .first();
      if (level) {
        await ctx.db.patch(level._id, {
          on_hand: level.on_hand - line.qty,
          updated_at: now,
        });
      } else {
        await ctx.db.insert("pos_stock_levels", {
          inventory_sku_id: line.skuId,
          on_hand: -line.qty,
          updated_at: now,
        });
      }

      await logAudit(ctx, {
        actor_id: "system",
        action: "stock.sale_movement",
        entity_type: "pos_inventory_skus",
        entity_id: line.skuId,
        source: "system",
        metadata: { transaction_id: args.transactionId, qty: -line.qty },
      });
    }
  },

```ts
import { internalMutation } from "../_generated/server";
import { v } from "convex/values";
import { logAudit } from "../audit/internal";

/**
 * Atomically redeem a voucher for a transaction. Called from
 * transactions.internal._confirmPaid_internal — paid-only.
 *
 * Returns { overRedeemed: true } when used_count has already hit
 * max_redemptions (race loser). Caller sets VOUCHER_OVER_REDEEMED flag and
 * proceeds — the txn still completes, the loser just doesn't get the discount
 * line item written. Spec §"vouchers/" + ADR-010.
 *
 * Returns { alreadyRedeemed: true } if this transaction already has a
 * redemption row — idempotent re-fire on confirm-paid retry.
 *
 * Convex serializable transactions + optimistic concurrency handle two
 * concurrent _confirmPaid invocations: one retries with the updated used_count
 * before completing. No manual lock needed. Test exercises this race.
 */
export const _redeemVoucher_internal = internalMutation({
  args: {
    voucher_id: v.id("pos_vouchers"),
    transaction_id: v.id("pos_transactions"),
    code_snapshot: v.string(),
    discount_amount: v.number(),
  },
  handler: async (ctx, args): Promise<{ overRedeemed: boolean; alreadyRedeemed: boolean }> => {
    // Idempotent re-fire check (emulated UNIQUE on transaction_id)
    const existing = await ctx.db
      .query("pos_voucher_redemptions")
      .withIndex("by_transaction", (q) => q.eq("transaction_id", args.transaction_id))
      .first();
    if (existing) {
      return { overRedeemed: false, alreadyRedeemed: true };
    }

    const voucher = await ctx.db.get(args.voucher_id);
    if (!voucher) throw new Error("VOUCHER_NOT_FOUND");

    // Race-tolerant over-redeem check (ADR-010 — one redemption per txn but
    // shared cap across txns; loser sets VOUCHER_OVER_REDEEMED flag)
    if (
      voucher.max_redemptions != null &&
      voucher.used_count >= voucher.max_redemptions
    ) {
      await logAudit(ctx, {
        actor_id: "system",
        action: "voucher.over_redeemed",
        entity_type: "pos_vouchers", entity_id: args.voucher_id,
        source: "system",
        metadata: { transaction_id: args.transaction_id, code: args.code_snapshot },
      });
      return { overRedeemed: true, alreadyRedeemed: false };
    }

    // Write redemption row + increment used_count
    await ctx.db.insert("pos_voucher_redemptions", {
      voucher_id: args.voucher_id,
      transaction_id: args.transaction_id,
      code_snapshot: args.code_snapshot,
      discount_amount: args.discount_amount,
      redeemed_at: Date.now(),
    });
    await ctx.db.patch(args.voucher_id, { used_count: voucher.used_count + 1 });

    await logAudit(ctx, {
      actor_id: "system",
      action: "voucher.redeemed",
      entity_type: "pos_vouchers", entity_id: args.voucher_id,
      source: "system",
      metadata: {
        transaction_id: args.transaction_id, code: args.code_snapshot,
        discount_amount: args.discount_amount,
      },
    });

    return { overRedeemed: false, alreadyRedeemed: false };
  },
});

/**
 * Reactive map of inventory_sku_id → on_hand for ACTIVE SKUs only.
 * Consumed by:
 *   - useCart (frontend) for live cart validation
 *   - catalog.public.catalog query (cross-module sub via Convex query composition)
 *
 * Inactive SKUs are filtered out at the source so consumers don't accidentally
 * surface retired SKUs in the cart UI.
 */
export const getStockLevels = query({
  args: {},
  handler: async (ctx): Promise<Record<string, number>> => {
    const activeSkus = await ctx.db
      .query("pos_inventory_skus")
      .withIndex("by_active", (q) => q.eq("active", true))
      .collect();
    const activeSet = new Set(activeSkus.map((s) => s._id));

    const levels = await ctx.db.query("pos_stock_levels").collect();
    const result: Record<string, number> = {};
    for (const lvl of levels) {
      if (activeSet.has(lvl.inventory_sku_id)) {
        result[lvl.inventory_sku_id] = lvl.on_hand;
      }
    }
    return result;
  },
});

import { query } from "../_generated/server";
import { v } from "convex/values";

/**
 * Active, unexpired vouchers. Bundled into the offline catalog cache snapshot
 * by useCatalogCache (Task 19) so cart-build can apply vouchers offline.
 * Server re-validates at commitCart per ADR-009.
 */
export const getActiveVouchers = query({
  args: {},
  handler: async (ctx) => {
    const now = Date.now();
    const rows = await ctx.db
      .query("pos_vouchers")
      .withIndex("by_active_expires", (q) => q.eq("active", true))
      .collect();
    return rows.filter((v) => v.expires_at == null || v.expires_at > now);
  },
});

/**
 * Validate a voucher code against a cart subtotal. Used by:
 *   - routes/sale/voucher.tsx for live UX feedback
 *   - transactions.public.commitCart for server-side re-validation
 *
 * Discount math per ADR-024:
 *   percentage: floor(subtotal * value / 100)
 *   amount:     min(value, subtotal)
 * Both integer rupiah per ADR-015.
 *
 * Returns { valid: false, reason } for: not found / expired / inactive /
 * below min_cart_value. Does NOT check used_count <= max_redemptions here —
 * the race is resolved at _redeemVoucher_internal (loser gets VOUCHER_OVER_REDEEMED).
 */
export const validateVoucher = query({
  args: { code: v.string(), cartSubtotal: v.number() },
  handler: async (ctx, args): Promise<{
    valid: boolean;
    discountAmount: number;
    voucherId?: string;
    reason?: "NOT_FOUND" | "INACTIVE" | "EXPIRED" | "MIN_CART_VALUE";
  }> => {
    const v = await ctx.db
      .query("pos_vouchers")
      .withIndex("by_code", (q) => q.eq("code", args.code.toUpperCase()))
      .first();
    if (!v) return { valid: false, discountAmount: 0, reason: "NOT_FOUND" };
    if (!v.active) return { valid: false, discountAmount: 0, reason: "INACTIVE" };
    if (v.expires_at != null && v.expires_at <= Date.now()) {
      return { valid: false, discountAmount: 0, reason: "EXPIRED" };
    }
    if (v.min_cart_value != null && args.cartSubtotal < v.min_cart_value) {
      return { valid: false, discountAmount: 0, reason: "MIN_CART_VALUE" };
    }

    const discountAmount =
      v.type === "percentage"
        ? Math.floor((args.cartSubtotal * v.value) / 100)
        : Math.min(v.value, args.cartSubtotal);

    return { valid: true, discountAmount, voucherId: v._id };
  },
});

import { internalMutation, internalQuery } from "../_generated/server";
import { v } from "convex/values";
import { Id } from "../_generated/dataModel";
import { internal } from "../_generated/api";
import { logAudit } from "../audit/internal";
import { wibYear } from "../lib/time";
import { NEG_STOCK, VOUCHER_OVER_REDEEMED, withFlag } from "./flags";

/**
 * Pure helper (no writes): for a cart of {productId, qty}, expand to
 * inventory_sku_id requirements via pos_product_components, then check
 * projected on_hand. Returns true if ANY SKU would go negative.
 *
 * Multi-product-same-SKU correctness (staffreview T4): a cart with Dubai 1pc +
 * Dubai 8pc shares the "dubai" SKU. We sum qtys per SKU before the projected
 * check.
 */
export const _projectedNegStockFlag_internal = internalQuery({
  args: {
    lines: v.array(v.object({
      productId: v.id("pos_products"),
      qty: v.number(),
    })),
  },
  handler: async (ctx, args): Promise<boolean> => {
    const perSku: Record<string, number> = {};
    for (const { productId, qty } of args.lines) {
      const components = await ctx.db
        .query("pos_product_components")
        .withIndex("by_product", (q) => q.eq("product_id", productId))
        .collect();
      for (const c of components) {
        const skuId = c.inventory_sku_id as unknown as string;
        perSku[skuId] = (perSku[skuId] ?? 0) + c.qty * qty;
      }
    }
    for (const [skuIdStr, needed] of Object.entries(perSku)) {
      const lvl = await ctx.db
        .query("pos_stock_levels")
        .withIndex("by_sku", (q) =>
          q.eq("inventory_sku_id", skuIdStr as Id<"pos_inventory_skus">),
        )
        .first();
      const available = lvl?.on_hand ?? 0;
      if (available < needed) return true;
    }
    return false;
  },
});

/**
 * Atomic receipt-number allocator. Returns "R-{wibYear}-{NNNN}" where the
 * year is the WIB calendar year (UTC+7, no DST) — see staffreview Critical #2
 * + convex/lib/time.ts.
 *
 * Convex optimistic concurrency handles concurrent _confirmPaid calls in the
 * same year: one retries with the updated next_number.
 */
export const _allocateReceiptNumber_internal = internalMutation({
  args: {},
  handler: async (ctx): Promise<string> => {
    const year = wibYear(Date.now());
    const counter = await ctx.db
      .query("pos_receipt_counters")
      .withIndex("by_year", (q) => q.eq("year", year))
      .first();
    let n: number;
    if (counter) {
      n = counter.next_number;
      await ctx.db.patch(counter._id, { next_number: n + 1 });
    } else {
      n = 1;
      await ctx.db.insert("pos_receipt_counters", { year, next_number: 2 });
    }
    return `R-${year}-${String(n).padStart(4, "0")}`;
  },
});

/**
 * THE FUNNEL. All three confirmation paths (webhook, polling, manual) converge
 * here. Idempotent via status guard: if txn is not in awaiting_payment, this
 * is a no-op. Spec §"Canonical sale data flow → THE FUNNEL".
 *
 * Steps (in order):
 *   1. Status guard (idempotent re-fire)
 *   2. Allocate receipt_number (atomic counter increment)
 *   3. Record sale movements (paid-only stock decrement, ADR-026 dedup)
 *   4. Re-check NEG_STOCK at confirm time (stock may have drained between
 *      commit and confirm — set flag if so)
 *   5. Redeem voucher if present (race-tolerant — loser sets VOUCHER_OVER_REDEEMED)
 *   6. Patch txn: status=paid, paid_at, confirmed_via, mgr_approver_id?, manual_reason?
 *   7. Audit log payment.confirmed with source metadata
 */
export const _confirmPaid_internal = internalMutation({
  args: {
    txnId: v.id("pos_transactions"),
    source: v.union(v.literal("webhook"), v.literal("polling"), v.literal("manual")),
    mgr_approver_id: v.optional(v.id("staff")),
    manual_reason: v.optional(v.string()),
  },
  handler: async (ctx, args) => {
    const txn = await ctx.db.get(args.txnId);
    if (!txn) throw new Error("TXN_NOT_FOUND");
    if (txn.status !== "awaiting_payment") {
      // Idempotent re-fire — second confirmation path arriving after first wins.
      return;
    }

    // 1. Receipt number
    const receiptNumber = await ctx.runMutation(
      internal.transactions.internal._allocateReceiptNumber_internal,
      {},
    );

    // 2. Lines + SKU expansion for movements
    const lines = await ctx.db
      .query("pos_transaction_lines")
      .withIndex("by_transaction", (q) => q.eq("transaction_id", args.txnId))
      .collect();

    const movementLines: Array<{
      lineId: Id<"pos_transaction_lines">;
      skuId: Id<"pos_inventory_skus">;
      qty: number;
    }> = [];
    const perSku: Record<string, number> = {};
    for (const line of lines) {
      const components = await ctx.db
        .query("pos_product_components")
        .withIndex("by_product", (q) => q.eq("product_id", line.product_id))
        .collect();
      for (const c of components) {
        movementLines.push({
          lineId: line._id,
          skuId: c.inventory_sku_id,
          qty: c.qty * line.qty,
        });
        const k = c.inventory_sku_id as unknown as string;
        perSku[k] = (perSku[k] ?? 0) + c.qty * line.qty;
      }
    }

    // 3. Record movements (paid-only decrement, ADR-026 dedup)
    await ctx.runMutation(internal.inventory.internal._recordSaleMovement_internal, {
      transactionId: args.txnId, lines: movementLines,
    });

    // 4. Re-check NEG_STOCK after decrement — any SKU now < 0 sets flag
    let flags = txn.flags;
    for (const skuIdStr of Object.keys(perSku)) {
      const lvl = await ctx.db
        .query("pos_stock_levels")
        .withIndex("by_sku", (q) =>
          q.eq("inventory_sku_id", skuIdStr as Id<"pos_inventory_skus">),
        )
        .first();
      if ((lvl?.on_hand ?? 0) < 0) {
        flags = withFlag(flags, NEG_STOCK);
        break;
      }
    }

    // 5. Voucher redemption (if applicable)
    if (txn.voucher_code_snapshot && txn.voucher_discount > 0) {
      const voucher = await ctx.db
        .query("pos_vouchers")
        .withIndex("by_code", (q) => q.eq("code", txn.voucher_code_snapshot!))
        .first();
      if (voucher) {
        const r = await ctx.runMutation(internal.vouchers.internal._redeemVoucher_internal, {
          voucher_id: voucher._id,
          transaction_id: args.txnId,
          code_snapshot: txn.voucher_code_snapshot,
          discount_amount: txn.voucher_discount,
        });
        if (r.overRedeemed) flags = withFlag(flags, VOUCHER_OVER_REDEEMED);
      }
    }

    // 6. Patch txn → paid
    await ctx.db.patch(args.txnId, {
      status: "paid",
      receipt_number: receiptNumber,
      paid_at: Date.now(),
      confirmed_via: args.source,
      confirmed_mgr_approver_id: args.mgr_approver_id,
      confirmed_manual_reason: args.manual_reason,
      flags,
    });

    // 7. Audit
    await logAudit(ctx, {
      actor_id: args.mgr_approver_id ?? "system",
      action: "payment.confirmed",
      entity_type: "pos_transactions", entity_id: args.txnId,
      mgr_approver_id: args.mgr_approver_id,
      source: args.source === "manual" ? "booth_inline" : "system",
      reason: args.manual_reason,
      metadata: { source: args.source, receipt_number: receiptNumber },
    });
  },
});

/**
 * Commit a built cart to the server. Single mutation — no per-line server
 * calls (Zustand cart on client). intent="draft" → status=draft; intent="charge"
 * → status=awaiting_payment (frontend then calls payments.actions.requestPayment).
 *
 * Snapshots prices + names at commit time (ADR-001 immutable snapshots).
 * Computes voucher_discount via validateVoucher logic (re-validated server-side).
 * Sets NEG_STOCK flag if projected on_hand would go negative (ADR-018).
 */
export const commitCart = mutation({
  args: commitCartArgs,
  handler: withIdempotency<
    {
      sessionId: Id<"staff_sessions">;
      idempotencyKey: string;
      intent: "draft" | "charge";
      lines: Array<{ productId: Id<"pos_products">; qty: number }>;
      voucherCode?: string;
    },
    { transactionId: Id<"pos_transactions">; totals: { subtotal: number; discount: number; total: number }; flags: number }
  >(
    "transactions.commitCart",
    async (ctx, args) => {
      const { staffId, deviceId } = await resolveSessionStaff(ctx, args.sessionId);

      // Snapshot prices + names
      const linesWithSnapshot: Array<{
        productId: Id<"pos_products">;
        qty: number;
        unit_price: number;
        product_name: string;
        product_code: string;
        tax_rate: number;
      }> = [];
      let subtotal = 0;
      for (const { productId, qty } of args.lines) {
        const p = await ctx.db.get(productId);
        if (!p || !p.active) throw new Error("PRODUCT_NOT_FOUND_OR_INACTIVE");
        const lineSub = p.price_idr * qty;
        subtotal += lineSub;
        linesWithSnapshot.push({
          productId, qty,
          unit_price: p.price_idr,
          product_name: p.name,
          product_code: p.code ?? p.sku_family,
          tax_rate: p.tax_rate,
        });
      }

      // Voucher re-validation (ADR-009)
      let voucherDiscount = 0;
      let voucherCodeSnapshot: string | undefined;
      if (args.voucherCode) {
        const code = args.voucherCode.toUpperCase();
        const v = await ctx.db
          .query("pos_vouchers")
          .withIndex("by_code", (q) => q.eq("code", code))
          .first();
        if (
          v && v.active &&
          (v.expires_at == null || v.expires_at > Date.now()) &&
          (v.min_cart_value == null || subtotal >= v.min_cart_value)
        ) {
          voucherDiscount = v.type === "percentage"
            ? Math.floor((subtotal * v.value) / 100)
            : Math.min(v.value, subtotal);
          voucherCodeSnapshot = code;
        }
        // If voucher fails server-side, silently drop — frontend already
        // showed the user the discount via validateVoucher; failure here is
        // exceptional and recovery is "no discount", not "abort sale".
      }

      const total = subtotal - voucherDiscount;

      // NEG_STOCK projection (ADR-018, multi-product-same-SKU correct per staffreview T4)
      const flagged = await ctx.runQuery(
        internal.transactions.internal._projectedNegStockFlag_internal,
        { lines: args.lines },
      );
      let flags = 0;
      if (flagged) flags = withFlag(flags, NEG_STOCK);

      // Insert txn + lines
      const txnId = await ctx.db.insert("pos_transactions", {
        status: args.intent === "draft" ? "draft" : "awaiting_payment",
        subtotal,
        voucher_code_snapshot: voucherCodeSnapshot,
        voucher_discount: voucherDiscount,
        total,
        flags,
        staff_id: staffId,
        created_at: Date.now(),
      });
      for (const l of linesWithSnapshot) {
        await ctx.db.insert("pos_transaction_lines", {
          transaction_id: txnId,
          product_id: l.productId,
          product_code_snapshot: l.product_code,
          product_name_snapshot: l.product_name,
          unit_price_snapshot: l.unit_price,
          tax_rate_snapshot: l.tax_rate,
          qty: l.qty,
          line_subtotal: l.unit_price * l.qty,
        });
      }

      await logAudit(ctx, {
        actor_id: staffId,
        action: "transaction.committed",
        entity_type: "pos_transactions", entity_id: txnId,
        source: "booth_inline",
        device_id: deviceId,
        metadata: { intent: args.intent, total, voucher_code: voucherCodeSnapshot },
      });

      return {
        transactionId: txnId,
        totals: { subtotal, discount: voucherDiscount, total },
        flags,
      };
    },
    {
      staffIdFromArgs: () => undefined, // resolved inside handler
    },
  ),
});

const draftArgs = {
  sessionId: v.id("staff_sessions"),
  draftId: v.id("pos_transactions"),
  idempotencyKey: v.string(),
};

/**
 * Resume a draft: returns its lines + voucherCode, then DELETES the row.
 *
 * Race protection (staffreview T6): two concurrent resumeDraft calls on the
 * same draftId — Convex's serializable transactions ensure one wins; the
 * loser re-reads the now-deleted draft and throws DRAFT_ALREADY_RESUMED.
 */
export const resumeDraft = mutation({
  args: draftArgs,
  handler: withIdempotency<
    { sessionId: Id<"staff_sessions">; draftId: Id<"pos_transactions">; idempotencyKey: string },
    {
      lines: Array<{ productId: Id<"pos_products">; qty: number }>;
      voucherCode?: string;
    }
  >(
    "transactions.resumeDraft",
    async (ctx, args) => {
      const { staffId, deviceId } = await resolveSessionStaff(ctx, args.sessionId);
      const draft = await ctx.db.get(args.draftId);
      if (!draft) throw new Error("DRAFT_ALREADY_RESUMED");
      if (draft.status !== "draft") throw new Error("INVALID_DRAFT_STATE");

      const lines = await ctx.db
        .query("pos_transaction_lines")
        .withIndex("by_transaction", (q) => q.eq("transaction_id", args.draftId))
        .collect();
      const result = {
        lines: lines.map((l) => ({ productId: l.product_id, qty: l.qty })),
        voucherCode: draft.voucher_code_snapshot,
      };

      // Delete lines then draft
      for (const l of lines) await ctx.db.delete(l._id);
      await ctx.db.delete(args.draftId);

      await logAudit(ctx, {
        actor_id: staffId,
        action: "transaction.cancelled",
        entity_type: "pos_transactions", entity_id: args.draftId,
        source: "booth_inline",
        device_id: deviceId,
        reason: "resumed_to_cart",
      });

      return result;
    },
  ),
});

export const deleteDraft = mutation({
  args: draftArgs,
  handler: withIdempotency<
    { sessionId: Id<"staff_sessions">; draftId: Id<"pos_transactions">; idempotencyKey: string },
    { deleted: true }
  >(
    "transactions.deleteDraft",
    async (ctx, args) => {
      const { staffId, deviceId } = await resolveSessionStaff(ctx, args.sessionId);
      const draft = await ctx.db.get(args.draftId);
      if (!draft) return { deleted: true };
      if (draft.status !== "draft") throw new Error("INVALID_DRAFT_STATE");
      const lines = await ctx.db
        .query("pos_transaction_lines")
        .withIndex("by_transaction", (q) => q.eq("transaction_id", args.draftId))
        .collect();
      for (const l of lines) await ctx.db.delete(l._id);
      await ctx.db.delete(args.draftId);
      await logAudit(ctx, {
        actor_id: staffId,
        action: "transaction.cancelled",
        entity_type: "pos_transactions", entity_id: args.draftId,
        source: "booth_inline",
        device_id: deviceId,
        reason: "draft_deleted",
      });
      return { deleted: true as const };
    },
  ),
});

/**

 * payments.actions.requestPayment after the Xendit HTTP POST succeeds.

 * This resolves staffreview Critical #1: an action retry that gets past

 * no double Convex-side invoice persistence.

    idempotencyKey: v.string(),
    txnId: v.id("pos_transactions"),
    xendit_invoice_id: v.string(),
    xendit_idempotency_key: v.string(),
    method: v.union(v.literal("QRIS"), v.literal("BCA_VA")),
    qr_string: v.optional(v.string()),
    va_number: v.optional(v.string()),

"use node";

import { action } from "../_generated/server";
import { v } from "convex/values";
import { Id } from "../_generated/dataModel";
import { internal, api } from "../_generated/api";
import { argon2Verify } from "hash-wasm";

const XENDIT_BASE = "https://api.xendit.co";

async function xenditPost<T = any>(
  path: string,
  body: Record<string, unknown>,
  idempotencyKey: string,
): Promise<{ ok: boolean; status: number; data: T }> {
  const key = process.env.XENDIT_SECRET_KEY;
  if (!key) throw new Error("XENDIT_SECRET_KEY not set");
  const auth = `Basic ${Buffer.from(`${key}:`).toString("base64")}`;
  const r = await fetch(`${XENDIT_BASE}${path}`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: auth,
      "X-IDEMPOTENCY-KEY": idempotencyKey,
    },
    body: JSON.stringify(body),
  });
  return { ok: r.ok, status: r.status, data: await r.json() };
}

async function xenditGet<T = any>(path: string): Promise<{ ok: boolean; data: T }> {
  const key = process.env.XENDIT_SECRET_KEY;
  if (!key) throw new Error("XENDIT_SECRET_KEY not set");
  const auth = `Basic ${Buffer.from(`${key}:`).toString("base64")}`;
  const r = await fetch(`${XENDIT_BASE}${path}`, {
    method: "GET",
    headers: { Authorization: auth },
  });
  return { ok: r.ok, data: await r.json() };
}

/**
 * Create a Xendit invoice for a transaction. Action-level idempotency pattern
 * per spec §"Action-level idempotency" + staffreview Critical #1:
 *   1. Pre-check Convex cache via _lookup_internal. Hit → return cached, skip HTTP.
 *   2. POST to Xendit with X-IDEMPOTENCY-KEY = args.idempotencyKey.
 *   3. runMutation _persistInvoiceCommit_internal (which writes the cache row
 *      atomically with the invoice row).
 */
export const requestPayment = action({
  args: {
    sessionId: v.id("staff_sessions"),
    txnId: v.id("pos_transactions"),
    method: v.union(v.literal("QRIS"), v.literal("BCA_VA")),
    idempotencyKey: v.string(),
  },
  handler: async (ctx, args): Promise<{
    invoiceId: Id<"pos_xendit_invoices">;
    method: "QRIS" | "BCA_VA";
    qrString?: string;
    vaNumber?: string;
  }> => {
    // 1. Cache pre-check
    const cached = await ctx.runQuery(internal.idempotency.internal._lookup_internal, {
      key: args.idempotencyKey,
    });
    if (cached) return JSON.parse(cached);

    // 2. Resolve txn for amount + description
    const txn = await ctx.runQuery(api.transactions.public.getById, { txnId: args.txnId });
    if (!txn) throw new Error("TXN_NOT_FOUND");
    if (txn.status !== "awaiting_payment") throw new Error("INVALID_STATE");

    // 3. POST Xendit
    const payload: Record<string, unknown> = {
      external_id: `pos-${args.txnId}`,
      amount: txn.total,
      payment_methods: args.method === "QRIS" ? ["QRIS"] : ["BCA"],
      description: `Frollie POS sale ${args.txnId}`,
    };
    const { ok, data } = await xenditPost("/v2/invoices", payload, args.idempotencyKey);
    if (!ok) throw new Error(`XENDIT_INVOICE_FAILED: ${JSON.stringify(data)}`);

    // 4. Commit invoice + cache row atomically
    const { invoiceId } = await ctx.runMutation(
      internal.payments.internal._persistInvoiceCommit_internal,
      {
        idempotencyKey: args.idempotencyKey,
        txnId: args.txnId,
        xendit_invoice_id: data.id,
        xendit_idempotency_key: args.idempotencyKey,
        method: args.method,
        qr_string: data.qr_string,
        va_number: data.account_number,
        status_at_create: data.status,
      },
    );

    const response = {
      invoiceId,
      method: args.method,
      qrString: data.qr_string as string | undefined,
      vaNumber: data.account_number as string | undefined,
    };
    return response;
  },
});

/**
 * Retry with a fresh invoice — cancels the current invoice on Xendit
 * (best-effort) and creates a new one. Per ADR-014 single-active-invoice.
 */
export const retryWithFreshInvoice = action({
  args: {
    sessionId: v.id("staff_sessions"),
    txnId: v.id("pos_transactions"),
    method: v.union(v.literal("QRIS"), v.literal("BCA_VA")),
    idempotencyKey: v.string(),
  },
  handler: async (ctx, args): Promise<{
    invoiceId: Id<"pos_xendit_invoices">;
    method: "QRIS" | "BCA_VA";
    qrString?: string;
    vaNumber?: string;
  }> => {
    const cached = await ctx.runQuery(internal.idempotency.internal._lookup_internal, {
      key: args.idempotencyKey,
    });
    if (cached) return JSON.parse(cached);

    const prev = await ctx.runQuery(api.payments.public.getCurrentInvoice, { txnId: args.txnId });
    let cancel_outcome: { success: boolean; error?: string } = { success: true };
    if (prev) {
      try {
        const { ok, data } = await xenditPost(`/invoices/${prev.xendit_invoice_id}/expire!`, {}, `${args.idempotencyKey}:cancel`);
        if (!ok) cancel_outcome = { success: false, error: JSON.stringify(data) };
      } catch (e: any) {
        cancel_outcome = { success: false, error: String(e?.message ?? e) };
      }
    }

    const txn = await ctx.runQuery(api.transactions.public.getById, { txnId: args.txnId });
    if (!txn) throw new Error("TXN_NOT_FOUND");
    const { ok, data } = await xenditPost("/v2/invoices", {
      external_id: `pos-${args.txnId}-retry-${Date.now()}`,
      amount: txn.total,
      payment_methods: args.method === "QRIS" ? ["QRIS"] : ["BCA"],
      description: `Frollie POS sale ${args.txnId} (retry)`,
    }, args.idempotencyKey);
    if (!ok) throw new Error(`XENDIT_INVOICE_FAILED: ${JSON.stringify(data)}`);

    const { invoiceId } = await ctx.runMutation(
      internal.payments.internal._replaceInvoiceCommit_internal,
      {
        idempotencyKey: args.idempotencyKey,
        txnId: args.txnId,
        prev_invoice_id: prev!._id,
        new_xendit_id: data.id,
        new_xendit_idempotency_key: args.idempotencyKey,
        method: args.method,
        qr_string: data.qr_string,
        va_number: data.account_number,
        status_at_create: data.status,
        cancel_outcome,
      },
    );

    return {
      invoiceId,
      method: args.method,
      qrString: data.qr_string,
      vaNumber: data.account_number,
    };
  },
});

/**
 * Polling fallback — GET invoice status; if PAID, runMutation _onPaidPolling.
 * No idempotencyKey arg: this is a side-effect-free read for the PENDING case,
 * and the funnel is idempotent for the PAID case.
 */
export const checkInvoiceStatus = action({
  args: { invoiceId: v.string() },
  handler: async (ctx, args): Promise<{
    status: "PENDING" | "PAID" | "EXPIRED" | "UNKNOWN";
  }> => {
    const { ok, data } = await xenditGet(`/v2/invoices/${args.invoiceId}`);
    if (!ok) return { status: "UNKNOWN" };
    if (data.status === "PAID") {
      await ctx.runMutation(internal.payments.internal._onPaidPolling_internal, {
        xendit_invoice_id: args.invoiceId,
      });
      return { status: "PAID" };
    }
    if (data.status === "EXPIRED") return { status: "EXPIRED" };
    return { status: "PENDING" };
  },
});

  handler: async (ctx, args): Promise<{ confirmed: true; receiptNumber: string }> => {
    const cached = await ctx.runQuery(internal.idempotency.internal._lookup_internal, {
      key: args.idempotencyKey,

    const session = await ctx.runQuery(api.auth.public.getSession, { sessionId: args.sessionId });
    if (!session) throw new Error("SESSION_INVALID");

    // for any staff. v0.3: require sessionStaff.role==="manager" — booth has
    // a manager present. v0.4 will route through approvals/ for off-booth.)
    const actor = await ctx.runQuery(internal.auth.internal._getStaffPinHash_internal, {
      staffId: session.staff_id,
    });

    const ok = await argon2Verify({ password: args.managerPin, hash: actor.pin_hash });
    if (!ok) {
      await ctx.runMutation(internal.auth.internal._recordFailedAttempt_internal, {
        idempotencyKey: `${args.idempotencyKey}:failed`,
        staffId: session.staff_id,
        deviceId: session.device_id,
      });
      throw new Error("INVALID_PIN");
    }

    // Commit funnel
    await ctx.runMutation(internal.payments.internal._onPaidManual_internal, {
      txnId: args.txnId, reason: args.reason, mgr_approver_id: session.staff_id,
    });

    // Read back receipt_number to return
    const txn = await ctx.runQuery(api.transactions.public.getById, { txnId: args.txnId });

    // Write the idempotency cache via a one-off internal mutation (no state to
    // commit — the funnel already wrote everything). Use a thin wrapper:

    await ctx.runMutation(internal.payments.internal._cacheManualResponse_internal, {
      idempotencyKey: args.idempotencyKey,
      response: JSON.stringify(response),
    });

  },
});

import { httpAction } from "../_generated/server";
import { internal } from "../_generated/api";

const webhookBodyValidator = (body: unknown): body is {
  id: string;
  status: "PAID" | "EXPIRED" | "PENDING";
  external_id?: string;
} => {
  if (typeof body !== "object" || body === null) return false;
  const b = body as Record<string, unknown>;
  return typeof b.id === "string" && typeof b.status === "string";
};

/**
 * Inbound Xendit webhook. Signature-verified via constant-time compare of the
 * x-callback-token header against XENDIT_CALLBACK_TOKEN env var.
 *
 * Dedup via _onPaidWebhook_internal → _confirmPaid_internal status guard.
 * Always returns 200 on success/duplicate (Xendit retries on non-2xx).
 */
export const xenditWebhook = httpAction(async (ctx, request) => {
  const expected = process.env.XENDIT_CALLBACK_TOKEN;
  if (!expected) return new Response("misconfigured", { status: 500 });
  const received = request.headers.get("x-callback-token") ?? "";
  // Constant-time string compare
  if (received.length !== expected.length) return new Response("unauthorized", { status: 401 });
  let diff = 0;
  for (let i = 0; i < received.length; i++) diff |= received.charCodeAt(i) ^ expected.charCodeAt(i);
  if (diff !== 0) return new Response("unauthorized", { status: 401 });

  const body = await request.json();
  if (!webhookBodyValidator(body)) return new Response("bad request", { status: 400 });

  if (body.status === "PAID") {
    await ctx.runMutation(internal.payments.internal._onPaidWebhook_internal, {
      xendit_invoice_id: body.id,
    });
  }
  // PENDING / EXPIRED → ignore (status flips happen via polling or expiry handler in v0.5)
  return new Response("ok", { status: 200 });
});

 *   - Guard: txn.status must be "awaiting_payment".
 *   - Best-effort Xendit cancel HTTP if invoice exists; failure does NOT block.
 *   - runMutation _cancelCommit_internal (flips status, audit).

export const cancelTransaction = action({
  args: {
    sessionId: v.id("staff_sessions"),
    txnId: v.id("pos_transactions"),
    reason: v.string(),
    idempotencyKey: v.string(),
  },
  handler: async (ctx, args): Promise<{ cancelled: true }> => {
    const cached = await ctx.runQuery(internal.idempotency.internal._lookup_internal, {
      key: args.idempotencyKey,
    });
    if (cached) return JSON.parse(cached);

    const session = await ctx.runQuery(api.auth.public.getSession, { sessionId: args.sessionId });
    if (!session) throw new Error("SESSION_INVALID");

    const txn = await ctx.runQuery(api.transactions.public.getById, { txnId: args.txnId });
    if (!txn) throw new Error("TXN_NOT_FOUND");

    let cancel_outcome: { success: boolean; error?: string } | undefined;
    if (txn.xendit_invoice_id_current) {
      try {
        const xkey = process.env.XENDIT_SECRET_KEY;
        if (!xkey) throw new Error("XENDIT_SECRET_KEY not set");
        const auth = `Basic ${Buffer.from(`${xkey}:`).toString("base64")}`;
        const r = await fetch(
          `https://api.xendit.co/invoices/${txn.xendit_invoice_id_current}/expire!`,
          {
            method: "POST",
            headers: {
              Authorization: auth,
              "X-IDEMPOTENCY-KEY": `${args.idempotencyKey}:cancel`,
            },
          },
        );
        cancel_outcome = { success: r.ok };
        if (!r.ok) cancel_outcome.error = `Xendit ${r.status}`;
      } catch (e: any) {
        cancel_outcome = { success: false, error: String(e?.message ?? e) };
      }
    }

    await ctx.runMutation(internal.transactions.internal._cancelCommit_internal, {
      txnId: args.txnId,
      reason: args.reason,
      actor_staff_id: session.staff_id,
      device_id: session.device_id,
    });

    if (cancel_outcome !== undefined) {
      await ctx.runMutation(internal.payments.internal._auditInvoiceCancelOutcome_internal, {
        txnId: args.txnId,
        outcome: cancel_outcome,
      });
    }

    await ctx.runMutation(internal.payments.internal._cacheManualResponse_internal, {
      idempotencyKey: args.idempotencyKey,
      response: JSON.stringify({ cancelled: true }),
    });

    staffId: v.id("staff"),

/**
 * Self-change PIN. Spec §"PIN management → Self-change".
 *   1. Cache pre-check.
 *   2. argon2-verify currentPin against caller's pin_hash.
 *   3. Validate newPin (4 digits, !== currentPin).
 *   4. argon2-hash newPin.
 *   5. runMutation _changePinCommit_internal with actor=self.
 */
export const changePin = action({
  args: {
    sessionId: v.id("staff_sessions"),
    currentPin: v.string(),
    newPin: v.string(),
    idempotencyKey: v.string(),
  },
  handler: async (ctx, args): Promise<{ changed: true }> => {
    const cached = await ctx.runQuery(internal.idempotency.internal._lookup_internal, {
      key: args.idempotencyKey,
    });
    if (cached) return JSON.parse(cached);

    if (!/^\d{4}$/.test(args.newPin)) throw new Error("NEW_PIN_INVALID");
    if (args.currentPin === args.newPin) throw new Error("SAME_PIN");

    const session = await ctx.runQuery(api.auth.public.getSession, { sessionId: args.sessionId });
    if (!session) throw new Error("SESSION_INVALID");
    const staff = await ctx.runQuery(internal.auth.internal._getStaffPinHash_internal, {
      staffId: session.staff_id,
    });
    if (!staff || !staff.active) throw new Error("INVALID_PIN");

    const ok = await argon2Verify({ password: args.currentPin, hash: staff.pin_hash });
    if (!ok) {
      await ctx.runMutation(internal.auth.internal._recordFailedAttempt_internal, {
        idempotencyKey: `${args.idempotencyKey}:failed`,
        staffId: session.staff_id, deviceId: session.device_id,
      });
      throw new Error("INVALID_PIN");
    }

    const newHash = await ctx.runAction(internal.auth.actions._hashPin_internal, {
      pin: args.newPin,
    });
    await ctx.runMutation(internal.auth.internal._changePinCommit_internal, {
      staffId: session.staff_id, newPinHash: newHash, actor: { kind: "self" },
    });

    await ctx.runMutation(internal.payments.internal._cacheManualResponse_internal, {
      idempotencyKey: args.idempotencyKey,
      response: JSON.stringify({ changed: true }),
    });
    return { changed: true };
  },
});

/**
 * Manager-reset booth-inline. Spec §"PIN management → Manager-reset booth-inline".
 * Verify caller is a manager → argon2-verify managerPin against MANAGER's hash
 * (not target's) → hash newPin → commit funnel.
 */
export const resetStaffPin = action({
  args: {
    sessionId: v.id("staff_sessions"),
    targetStaffId: v.id("staff"),
    newPin: v.string(),
    managerPin: v.string(),
    idempotencyKey: v.string(),
  },
  handler: async (ctx, args): Promise<{ reset: true }> => {
    const cached = await ctx.runQuery(internal.idempotency.internal._lookup_internal, {
      key: args.idempotencyKey,
    });
    if (cached) return JSON.parse(cached);

    if (!/^\d{4}$/.test(args.newPin)) throw new Error("NEW_PIN_INVALID");

    const session = await ctx.runQuery(api.auth.public.getSession, { sessionId: args.sessionId });
    if (!session) throw new Error("SESSION_INVALID");
    const manager = await ctx.runQuery(internal.auth.internal._getStaffPinHash_internal, {
      staffId: session.staff_id,
    });
    if (!manager || !manager.active || manager.role !== "manager") throw new Error("NOT_MANAGER");
    if (session.staff_id === args.targetStaffId) {
      throw new Error("USE_CHANGE_PIN_FOR_SELF");
    }

    const target = await ctx.runQuery(internal.auth.internal._getStaffPinHash_internal, {
      staffId: args.targetStaffId,
    });
    if (!target || !target.active) throw new Error("TARGET_NOT_FOUND");

    const ok = await argon2Verify({ password: args.managerPin, hash: manager.pin_hash });
    if (!ok) {
      await ctx.runMutation(internal.auth.internal._recordFailedAttempt_internal, {
        idempotencyKey: `${args.idempotencyKey}:failed`,
        staffId: session.staff_id, deviceId: session.device_id,
      });
      throw new Error("INVALID_PIN");
    }

    const newHash = await ctx.runAction(internal.auth.actions._hashPin_internal, {
      pin: args.newPin,
    });
    await ctx.runMutation(internal.auth.internal._changePinCommit_internal, {
      staffId: args.targetStaffId, newPinHash: newHash,
      actor: { kind: "manager_reset", mgr_approver_id: session.staff_id },
    });

    await ctx.runMutation(internal.payments.internal._cacheManualResponse_internal, {
      idempotencyKey: args.idempotencyKey,
      response: JSON.stringify({ reset: true }),
    });
    return { reset: true };
  },
});
```

/**
 * Single payload for the catalog screen + offline cache. The active stock
 * levels live in the inventory module (ADR-034); v0.3 reads them via the
 * inventory module's query rather than crossing module boundaries.
 *
 * available_qty per product is still computed client-side (ADR-017) from
 * components + stockLevels.
 *
 * v0.3 adds vouchers to the cache snapshot (ADR-009 — vouchers are static,
 * managers create via Convex dashboard for now; offline apply must work).
 */
export const catalog = query({
  args: {},
  handler: async (ctx) => {
    const [products, skus, allComponents, allStockLevels, vouchers] = await Promise.all([
      ctx.db
        .query("pos_products")
        .withIndex("by_active_sort", (q) => q.eq("active", true))
        .collect(),
      ctx.db
        .query("pos_inventory_skus")
        .withIndex("by_active", (q) => q.eq("active", true))
        .collect(),
      ctx.db.query("pos_product_components").collect(),
      ctx.db.query("pos_stock_levels").collect(),
      ctx.db
        .query("pos_vouchers")
        .withIndex("by_active_expires", (q) => q.eq("active", true))
        .collect(),
    ]);

    const activeProductIds = new Set(products.map((p) => p._id));
    const activeSkuIds = new Set(skus.map((s) => s._id));

    const components = allComponents.filter((c) => activeProductIds.has(c.product_id));
    const stockLevels = allStockLevels.filter((s) => activeSkuIds.has(s.inventory_sku_id));
    const now = Date.now();
    const activeVouchers = vouchers.filter((v) => v.expires_at == null || v.expires_at > now);

    return { products, skus, components, stockLevels, vouchers: activeVouchers };
  },
});
```

import { query } from "../_generated/server";
import { v } from "convex/values";
import { createHash } from "node:crypto";

/**
 * Public projection of an approval request by RAW token. The raw token comes
 * from the URL bar; we sha256-hash it to look up by_token_hash index.
 *
 * Returns ONLY the fields needed for UI rendering (staffreview Refinement).
 * Never includes token_hash in the response — defense-in-depth: don't ship
 * even the hashed form back to the client.
 *
 * Computes effective status: "pending" requests past their expiry surface as
 * "expired" so the UI doesn't let a manager waste a PIN attempt on a stale link.
 * The DB row stays "pending" until a cleanup job (v0.5) sweeps it.
 */
export const getByToken = query({
  args: { rawToken: v.string() },
  handler: async (ctx, args): Promise<{
    kind: "staff_pin_reset";
    subject_staff_name: string;
    subject_staff_code?: string;
    status: "pending" | "resolved" | "expired";
    triggered_at: number;
    token_expires_at: number;
    resolved_at?: number;
  } | null> => {
    const hash = createHash("sha256").update(args.rawToken).digest("hex");
    const req = await ctx.db
      .query("pos_approval_requests")
      .withIndex("by_token_hash", (q) => q.eq("token_hash", hash))
      .first();
    if (!req) return null;
    const staff = await ctx.db.get(req.subject_staff_id);
    if (!staff) return null;

    let status: "pending" | "resolved" | "expired" = req.status;
    if (status === "pending" && req.token_expires_at <= Date.now()) {
      status = "expired";
    }

    return {
      kind: req.kind,
      subject_staff_name: staff.name,
      subject_staff_code: staff.code,
      status,
      triggered_at: req.triggered_at,
      token_expires_at: req.token_expires_at,
      resolved_at: req.resolved_at,
    };
  },
});

Two design points worth noting:
1. `approveStaffPinReset` needs to identify which manager is approving. Two options: (a) require the manager to identify themselves by their staff code in the form, OR (b) any manager who knows a manager PIN can approve and we look up by PIN-match. Per spec §"approveStaffPinReset" line 564 — "argon2id-verify managerPin against MANAGER's `pin_hash`" — implies (a). The approve form takes `managerStaffCode` so we know whose hash to verify against. ```ts
"use node";

import { internalAction, action } from "../_generated/server";
import { v } from "convex/values";
import { internal, api } from "../_generated/api";
import { argon2Verify } from "hash-wasm";
import { createHash, randomBytes, timingSafeEqual } from "node:crypto";

const TOKEN_TTL_MS = 60 * 60 * 1000; // 60 min per ADR-029

function sha256Hex(s: string): string {
  return createHash("sha256").update(s).digest("hex");
}

/**
 * Fired by auth/internal._recordFailedAttempt scheduler when a staff hits the
 * 3rd failed PIN attempt. Spec §"Telegram approval flow → staff_pin_reset".
 *
 * Dedup (staffreview Improvement #5): if a pending+unexpired request already
 * exists for this staff, skip silently.
 */
export const notifyStaffLockout = internalAction({
  args: { staffId: v.id("staff") },
  handler: async (ctx, args): Promise<{ skipped?: boolean; reason?: string }> => {
    // Dedup
    const existing = await ctx.runQuery(internal.approvals.internal._listPendingForStaff_internal, {
      staffId: args.staffId,
    });
    if (existing.length > 0) {
      return { skipped: true, reason: "pending_request_exists" };
    }

    const staff = await ctx.runQuery(internal.auth.internal._getStaffPinHash_internal, {
      staffId: args.staffId,
    });
    if (!staff) throw new Error("STAFF_NOT_FOUND");

    // Generate raw token (32 bytes URL-safe base64)
    const rawToken = randomBytes(32).toString("base64url");
    const tokenHash = sha256Hex(rawToken);
    const now = Date.now();

    const { requestId } = await ctx.runMutation(
      internal.approvals.internal._createRequest_internal,
      {
        kind: "staff_pin_reset",
        subject_staff_id: args.staffId,
        triggered_by_event: "auth_lockout",
        triggered_at: now,
        token_hash: tokenHash,
        token_expires_at: now + TOKEN_TTL_MS,
      },
    );

    const baseUrl = process.env.POS_BASE_URL;
    if (!baseUrl) throw new Error("POS_BASE_URL not set");
    const requestUrl = `${baseUrl}/approve/${rawToken}`;

    const staffRow = await ctx.runQuery(api.staff.public.getById, { staffId: args.staffId });
    await ctx.runAction(internal.telegram.send.sendTemplate, {
      kind: "staff_pin_reset",
      payload: {
        staff_name: staffRow?.name ?? "(unknown)",
        staff_code: staffRow?.code ?? "(no code)",
        locked_at_iso: new Date(now).toISOString(),
        request_url: requestUrl,
      },
    });

    await ctx.runMutation(internal.approvals.internal._markNotified_internal, { requestId });
    return {};
  },
});

/**
 * Off-booth PIN reset via Telegram link. Spec §"approveStaffPinReset".
 *   1. sha256 the raw token, look up request via by_token_hash, constant-time
 *      compare hashes (staffreview Improvement #7).
 *   2. Check status===pending AND token_expires_at > now.
 *   3. Look up manager by staff code.
 *   4. argon2-verify managerPin against MANAGER's hash.
 *   5. argon2-hash newPin.
 *   6. runMutation auth.internal._changePinCommit_internal with
 *      actor={kind:"manager_reset", mgr_approver_id}.
 *   7. runMutation approvals.internal._markResolved_internal.
 *
 * Locked-out manager self-reset (staffreview Improvement #6): intentional.
 * argon2Verify only consults pin_hash; lockout state is NOT checked here.
 */
export const approveStaffPinReset = action({
  args: {
    token: v.string(),
    managerPin: v.string(),
    newPin: v.string(),
    managerStaffCode: v.string(),
    idempotencyKey: v.string(),
  },
  handler: async (ctx, args): Promise<{ resolved: true }> => {
    const cached = await ctx.runQuery(internal.idempotency.internal._lookup_internal, {
      key: args.idempotencyKey,
    });
    if (cached) return JSON.parse(cached);

    if (!/^\d{4}$/.test(args.newPin)) throw new Error("NEW_PIN_INVALID");

    const tokenHash = sha256Hex(args.token);
    const req = await ctx.runQuery(internal.approvals.internal._getByTokenHash_internal, {
      tokenHash,
    });
    if (!req) throw new Error("TOKEN_INVALID");

    // Constant-time compare (staffreview Improvement #7) — defense-in-depth
    const a = Buffer.from(tokenHash, "hex");
    const b = Buffer.from(req.token_hash, "hex");
    if (a.length !== b.length || !timingSafeEqual(a, b)) {
      throw new Error("TOKEN_INVALID");
    }

    if (req.status !== "pending") throw new Error("REQUEST_RESOLVED");
    if (req.token_expires_at <= Date.now()) throw new Error("TOKEN_EXPIRED");

import { create } from "zustand";
import { persist, createJSONStorage } from "zustand/middleware";
import type { Id } from "../../convex/_generated/dataModel";

export interface CartLine {
  productId: Id<"pos_products">;
  qty: number;
  unitPrice: number; // for live subtotal — server snapshots at commit
}

interface CartState {
  lines: CartLine[];
  voucherCode?: string;
  addLine: (productId: Id<"pos_products">, unitPrice: number) => void;
  setQty: (productId: Id<"pos_products">, qty: number) => void;
  setVoucher: (code: string | undefined) => void;
  clearVoucher: () => void;
  loadFromDraft: (data: { lines: CartLine[]; voucherCode?: string }) => void;
  clear: () => void;
  subtotal: number;
}

/**
 * Client-only cart state. Spec §"Frontend hooks → useCart" + brainstorm record:
 * the cart builds entirely client-side and commits once via commitCart on
 * Save/Charge. No per-line server mutations.
 *
 * Persisted to sessionStorage: F5 keeps the cart; closing the tab discards it.
 * A second tab gets a fresh cart (sessionStorage is per-tab) — intentional to
 * avoid accidental cross-tab interference.
 *
 * Derived totals (`subtotal`) are computed on every change. Voucher discount
 * is computed by the SCREEN (route) via Convex validateVoucher reactive sub,
 * not stored here — that way the discount stays live as catalog/voucher data
 * changes underneath.
 */
export const useCart = create<CartState>()(
  persist(
    (set, get) => ({
      lines: [],
      voucherCode: undefined,
      get subtotal() {
        return get().lines.reduce((sum, l) => sum + l.qty * l.unitPrice, 0);
      },
      addLine: (productId, unitPrice) =>
        set((s) => {
          const existing = s.lines.find((l) => l.productId === productId);
          if (existing) {
            return {
              lines: s.lines.map((l) =>
                l.productId === productId ? { ...l, qty: l.qty + 1 } : l,
              ),
            };
          }
          return { lines: [...s.lines, { productId, qty: 1, unitPrice }] };
        }),
      setQty: (productId, qty) =>
        set((s) => ({
          lines:
            qty <= 0
              ? s.lines.filter((l) => l.productId !== productId)
              : s.lines.map((l) =>
                  l.productId === productId ? { ...l, qty } : l,
                ),
        })),
      setVoucher: (code) => set({ voucherCode: code }),
      clearVoucher: () => set({ voucherCode: undefined }),
      loadFromDraft: (data) =>
        set({ lines: data.lines, voucherCode: data.voucherCode }),
      clear: () => set({ lines: [], voucherCode: undefined }),
    }),
    {
      name: "frollie-cart",
      storage: createJSONStorage(() => sessionStorage),
      // Don't persist computed `subtotal` — it's recomputed on read.
      partialize: (s) => ({ lines: s.lines, voucherCode: s.voucherCode }),
    },
  ),
);
```

```ts
import { useEffect, useState } from "react";
import { openDB, type IDBPDatabase } from "idb";

const DB_NAME = "frollie-idem";
const STORE = "keys";
const TTL_MS = 24 * 60 * 60 * 1000;

let dbPromise: Promise<IDBPDatabase> | null = null;
function getDb() {
  if (!dbPromise) {
    dbPromise = openDB(DB_NAME, 1, {
      upgrade(db) {
        if (!db.objectStoreNames.contains(STORE)) db.createObjectStore(STORE);
      },
    });
  }
  return dbPromise;
}

/** Test-only — resets the cached connection. */
export function __resetForTests() {
  dbPromise = null;
}

interface IntentRow {
  key: string;
  expires_at: number;
}

async function getOrCreate(intent: string): Promise<string> {
  const db = await getDb();
  const existing = (await db.get(STORE, intent)) as IntentRow | undefined;
  if (existing && existing.expires_at > Date.now()) {
    return existing.key;
  }
  const fresh = `${intent}:${crypto.randomUUID()}`;
  await db.put(STORE, { key: fresh, expires_at: Date.now() + TTL_MS } satisfies IntentRow, intent);
  return fresh;
}

export async function clearIntent(intent: string): Promise<void> {
  const db = await getDb();
  await db.delete(STORE, intent);
}

/**
 * IDB-persisted idempotency keys with 24h TTL matching the server window
 * (ADR-013). Spec §"Frontend hooks → useIdempotency (UPDATE)".
 *
 * Returns undefined on first render (IDB read is async); useEffect resolves
 * and re-renders with the actual key. Callers MUST guard mutation calls on
 * `if (!key) return;` to avoid firing without a key.
 *
 * Why we changed from v0.2's in-memory pattern: reload-mid-payment must
 * dedupe at the server. Without IDB persistence, F5 mid-charge created a
 * fresh key → server saw it as a new mutation → double charge.
 */
export function useIdempotency(intent: string): string | undefined {
  const [key, setKey] = useState<string | undefined>(undefined);
  useEffect(() => {
    let cancelled = false;
    getOrCreate(intent).then((k) => {
      if (!cancelled) setKey(k);
    });
    return () => { cancelled = true; };
  }, [intent]);
  return key;
}

interface QueuedDraft {
  idempotencyKey: string;
  sessionId: Id<"staff_sessions">;
  lines: Array<{ productId: Id<"pos_products">; qty: number }>;
  voucherCode?: string;

 *
 * Each item is keyed by idempotencyKey — re-enqueue of the same draft is a
 * no-op. Server-side withIdempotency completes the dedup if a flush hits
 * before the original online path did.
 */
export function useOfflineQueue(
  executor: (draft: QueuedDraft) => Promise<Id<"pos_transactions">>,
) {
  const [pending, setPending] = useState(0);

  const refresh = useCallback(async () => {
    const db = await getDb();
    const count = await db.count(STORE);
    setPending(count);
  }, []);

  useEffect(() => { refresh(); }, [refresh]);

  const enqueue = useCallback(
    async (item: Omit<QueuedDraft, "enqueued_at">) => {
      const db = await getDb();
      await db.put(STORE, { ...item, enqueued_at: Date.now() } satisfies QueuedDraft);
      await refresh();
    },
    [refresh],
  );

  const flush = useCallback(async () => {
    const db = await getDb();
    const items = (await db.getAll(STORE)) as QueuedDraft[];
    const results = await Promise.allSettled(items.map(async (it) => {
      await executor(it);
      await db.delete(STORE, it.idempotencyKey);
    }));
    await refresh();
    if (results.some((r) => r.status === "rejected")) {
      throw new Error("Some queued items failed to flush");
    }
  }, [executor, refresh]);

  return { pending, enqueue, flush };
}
```

```ts
import { useEffect, useReducer } from "react";
import { useQuery, useAction } from "convex/react";
import { api } from "../../convex/_generated/api";
import type { Id } from "../../convex/_generated/dataModel";

type Phase =
  | { kind: "loading" }
  | { kind: "showing"; subPhase: "fresh" | "polling" | "ceiling_reached" }
  | { kind: "paid" }
  | { kind: "expired" }
  | { kind: "cancelled" };

const POLL_INTERVAL_MS = 2_000;
const POLL_CEILING_MS = 60_000;

export function useXenditPayment(txnId: Id<"pos_transactions">) {
  const txn = useQuery(api.transactions.public.getById, { txnId });
  const invoice = useQuery(api.payments.public.getCurrentInvoice, { txnId });
  const checkStatus = useAction(api.payments.actions.checkInvoiceStatus);

  // Compute the phase from reactive state
  const phase: Phase = computePhase(txn, invoice);

  useEffect(() => {
    if (!txn || !invoice) return;
    if (txn.status !== "awaiting_payment") return; // paid/cancelled → no polling

    const startedAt = Date.now();
    const handle = setInterval(async () => {
      // Stop if past ceiling — UI shows "ceiling_reached" state
      if (Date.now() - startedAt > POLL_CEILING_MS) {
        clearInterval(handle);
        return;
      }
      try {
        await checkStatus({ invoiceId: invoice.xendit_invoice_id });
      } catch (e) {
        // Network error — polling continues on next tick
      }
    }, POLL_INTERVAL_MS);

    // First poll fires after initial 2s wait (matches spec)
    const initial = setTimeout(() => {
      checkStatus({ invoiceId: invoice.xendit_invoice_id }).catch(() => {});
    }, POLL_INTERVAL_MS);

    return () => {
      clearInterval(handle);
      clearTimeout(initial);
    };
  }, [txn?.status, invoice?.xendit_invoice_id, checkStatus]);

  return { phase, invoice, txn };
}

function computePhase(
  txn: any | null | undefined,
  invoice: any | null | undefined,
): Phase {
  if (!txn || !invoice) return { kind: "loading" };
  if (txn.status === "paid") return { kind: "paid" };
  if (txn.status === "cancelled") return { kind: "cancelled" };
  // awaiting_payment — UI shows the QR/VA; subPhase is timing-driven and
  // managed by the route component, not this hook (keeps the hook simple).
  return { kind: "showing", subPhase: "fresh" };
}
```

The route component (Task 34) tracks the ceiling timing separately to drive UI affordances ("Manager override" CTA appears at 60s). Keep the hook focused on starting/stopping the polling.

import { useEffect, useRef } from "react";
import { useQuery, useAction } from "convex/react";
import { api } from "../../convex/_generated/api";
import { toast } from "sonner";
import type { Id } from "../../convex/_generated/dataModel";

/**
 * ADR-026: on startup, sweep awaiting_payment txns from the last 5 minutes
 * and re-check each invoice's Xendit status. Webhook race protected by
 * by_line_and_sku dedup index on movements.
 *
 * Mounted on RootLayout once the session is resolved. Idempotent — runs once
 * per mount (a ref guard prevents repeat-fires on transient re-renders).
 */
export function useStartupReconciliation(sessionId: Id<"staff_sessions"> | undefined) {
  const ran = useRef(false);
  const recent = useQuery(
    api.transactions.public.listRecentAwaitingPayment,
    sessionId ? { sessionId } : "skip",
  );
  const checkStatus = useAction(api.payments.actions.checkInvoiceStatus);

  useEffect(() => {
    if (ran.current) return;
    if (!recent) return;
    if (recent.length === 0) {
      ran.current = true;
      return;
    }
    ran.current = true;
    (async () => {
      let confirmed = 0;
      for (const txn of recent) {
        if (!txn.xendit_invoice_id_current) continue;
        try {
          const r = await checkStatus({ invoiceId: txn.xendit_invoice_id_current });
          if (r.status === "PAID") confirmed++;
        } catch { /* swallow, will retry on next reload */ }
      }
      if (confirmed > 0) {
        toast.success(`Reconciled ${confirmed} pending payment${confirmed > 1 ? "s" : ""}.`);
      }
    })();
  }, [recent, checkStatus]);
}

import { useNavigate } from "react-router-dom";
import { useMutation, useQuery } from "convex/react";
import { api } from "../../../convex/_generated/api";
import { useCart } from "@/hooks/useCart";
import { useSession } from "@/hooks/useSession";
import { useIdempotency } from "@/hooks/useIdempotency";
import { useCatalogCache } from "@/hooks/useCatalogCache";
import { Button } from "@/components/ui/button";
import { rp } from "@/lib/format";
import { toast } from "sonner";
import { NEG_STOCK, hasFlag } from "../../../convex/transactions/flags";

export default function Sale() {
  const nav = useNavigate();
  const session = useSession();
  const cart = useCart();
  const live = useQuery(api.catalog.public.catalog);
  const snapshot = useCatalogCache(live);
  const products = snapshot?.products ?? [];
  const stockLevels = useQuery(api.inventory.public.getStockLevels);
  const commitCart = useMutation(api.transactions.public.commitCart);

  const idKeyDraft = useIdempotency(`commitDraft:${cart.lines.map((l) => l.productId).join(",")}`);
  const idKeyCharge = useIdempotency(`commitCharge:${cart.lines.map((l) => l.productId).join(",")}`);

  async function saveDraft() {
    if (!session || !idKeyDraft) return;
    try {
      await commitCart({
        sessionId: session._id, idempotencyKey: idKeyDraft, intent: "draft",
        lines: cart.lines.map((l) => ({ productId: l.productId, qty: l.qty })),
        voucherCode: cart.voucherCode,
      });
      cart.clear();
      toast.success("Draft saved");
      nav("/sale/drafts");
    } catch (e) {
      toast.error(`Save failed: ${String(e)}`);
    }
  }

  async function charge() {
    if (!session || !idKeyCharge) return;
    try {
      const r = await commitCart({
        sessionId: session._id, idempotencyKey: idKeyCharge, intent: "charge",
        lines: cart.lines.map((l) => ({ productId: l.productId, qty: l.qty })),
        voucherCode: cart.voucherCode,
      });
      if (hasFlag(r.flags, NEG_STOCK)) {
        toast.warning("Sale flagged for negative stock — manager review needed.");
      }
      cart.clear();
      nav(`/sale/charge/${r.transactionId}`);
    } catch (e) {
      toast.error(`Charge failed: ${String(e)}`);
    }
  }

  return (
    <div className="flex h-full">
      <section className="flex-1 grid grid-cols-2 gap-2 p-4 overflow-auto">
        {products.map((p: any) => (
          <button key={p._id} className="border rounded p-3 text-left"
            onClick={() => cart.addLine(p._id, p.price_idr)}>
            <div className="font-medium">{p.name}</div>
            <div>{rp(p.price_idr)}</div>
          </button>
        ))}
      </section>
      <aside className="w-80 border-l p-4 flex flex-col">
        <div className="flex-1 overflow-auto">
          {cart.lines.map((l) => {
            const p = products.find((p: any) => p._id === l.productId);
            return (
              <div key={l.productId} className="flex justify-between items-center py-2">
                <div>{p?.name ?? l.productId}</div>
                <div>
                  <button onClick={() => cart.setQty(l.productId, l.qty - 1)}>−</button>
                  <span className="mx-2">{l.qty}</span>
                  <button onClick={() => cart.setQty(l.productId, l.qty + 1)}>+</button>
                </div>
              </div>
            );
          })}
        </div>
        <div className="py-2 border-t">
          <Button variant="link" onClick={() => nav("/sale/voucher")}>
            {cart.voucherCode ? `Voucher: ${cart.voucherCode}` : "+ Apply voucher"}
          </Button>
        </div>
        <div className="text-2xl font-bold py-2">{rp(cart.subtotal)}</div>
        <div className="flex gap-2">
          <Button variant="outline" onClick={saveDraft} disabled={cart.lines.length === 0}>
            Save draft
          </Button>
          <Button onClick={charge} disabled={cart.lines.length === 0}>
            Charge {rp(cart.subtotal)}
          </Button>
        </div>
      </aside>
    </div>
  );
}
