**Architecture:** Backend-heavy Convex changes across `auth/`, `transactions/`, `seed/`, `staff/`, `payments/`, `approvals/`, plus minimal FE threading (`useXenditPayment` session arg, a `must_change_pin` login prompt). Two schema additions (both backward-safe). One PR.

## Global Constraints

- **Money is integer rupiah** — no floats/cents (ADR-015).
- **`audit_log.action` is a free `v.string()`** — no enum to edit; document new verbs in `docs/SCHEMA.md`.
- **Server time wins** — every `_at` via `Date.now()` inside the function (ADR-031).
- **Public mutations:** keep `idempotencyKey` + `withIdempotency` + `authCheck`; authCheck runs before cache lookup (ADR-013/046). *Do not* remove idempotency from public mutations — SEC-01 only removes it from the internal failed-attempt counter.
- **Never log PIN values.**
- **`convex/lib/` must stay V8-safe** (no `"use node"`).
- **Tests:** `npx vitest run <path>` (edge-runtime env, `server.deps.inline` for convex-test — see `convex-test-vitest-config` setup). Payment/refund/stock/auth changes REQUIRE tests (CLAUDE.md "How to add a feature" #7).
- **Branch:** all work on `worktree-v1.1-security-hardening` (already created). Commit per task.
- **Pre-push:** `npm run typecheck` + `npx vitest run` green.

**Task order (independent unless noted):** 1 (SEC-02) → 2 (SEC-03) → 3 (SEC-01+07, atomic) → 4 (SEC-05+06, atomic) → 5 (SEC-04) → 6 (docs).

```ts
export const _recordFailedAttempt_internal = internalMutation({
  args: {
    staffId: v.id("staff"),
    deviceId: v.string(),
    // SEC-01: was an idempotencyKey-derived withIdempotency wrap (old "Fix 10"),
    // which let a reused client key freeze fail_count at 1 and defeat lockout.
    // Now keyed solely on staff_id; over-counting on a crash-retry is fail-SAFE.
    countTowardLockout: v.boolean(),
    source: v.optional(failedAttemptSourceValidator),
  },
  handler: async (ctx, args): Promise<{ newly_locked: boolean; seconds_remaining: number }> => {
    const now = Date.now();
    const source = args.source ?? "booth_inline";

    // SEC-07: off-booth Telegram approve misses are AUDITED (rule #10 trail) but
    // must NOT touch the booth lockout counter — else a leaked approval token
    // lets an attacker DoS-lock a manager's booth login. Per-token cap
    // (_recordTokenPinFailure_internal, 5) bounds brute force on that path.
    if (!args.countTowardLockout) {
      await logAudit(ctx, {
        actor_id: args.staffId, action: "staff.failed_pin",
        entity_type: "staff", entity_id: args.staffId, source, device_id: args.deviceId,
      });
      return { newly_locked: false, seconds_remaining: 0 };
    }

    const attempt = await cleanupAndGetAttempt(ctx, args.staffId);
    const lockExpired = attempt?.locked_until != null && attempt.locked_until <= now;
    const next = lockExpired ? 1 : (attempt?.fail_count ?? 0) + 1;
    const lock = next >= MAX_FAILS ? now + LOCKOUT_MS : null;
    if (attempt) {
      await ctx.db.patch(attempt._id, { fail_count: next, locked_until: lock, last_attempt_at: now });
    } else {
      await ctx.db.insert("pos_auth_attempts", {
        staff_id: args.staffId, fail_count: next, locked_until: lock, last_attempt_at: now,
      });
    }
    await logAudit(ctx, {
      actor_id: args.staffId, action: "staff.failed_pin",
      entity_type: "staff", entity_id: args.staffId, source, device_id: args.deviceId,
    });
    if (lock) {
      await logAudit(ctx, {
        actor_id: args.staffId, action: "staff.locked_out",
        entity_type: "staff", entity_id: args.staffId, source, device_id: args.deviceId,
        reason: `${MAX_FAILS} consecutive failures`,
      });
      await ctx.scheduler.runAfter(0, internal.approvals.actions.notifyStaffLockout, { staffId: args.staffId });
    }
    return { newly_locked: lock != null, seconds_remaining: lock ? Math.ceil((lock - now) / 1000) : 0 };
  },
});
```
*(Keep the `failedAttemptSourceValidator`, `cleanupAndGetAttempt`, `MAX_FAILS`, `LOCKOUT_MS`, `withIdempotency` import — `withIdempotency` is still used by `_loginCommit_internal` etc.)*
