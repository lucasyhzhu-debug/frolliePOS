## 2. Critical Issues (Must Fix)

| 1 | Badge cva test assertion is wrong (cva doesn't fall back to default for explicit unknown variants) | Testing | Task 5 Step 3 |

### Issue 1: `badgeVariants({variant:"gofood"}).toBe(badgeVariants({variant:"default"}))` is false
`cva` applies `defaultVariants` only when a prop is `undefined`. An **explicit** unknown value (`"gofood"`) contributes no classes but does **not** trigger the default — so it yields base-only, while `"default"` yields base + default classes. They are not equal; the assertion would fail at runtime. The type-level removal is what we actually want to guard, via `@ts-expect-error` (enforced by `npm run typecheck`, since vitest/esbuild ignores types).

**Recommendation:** replace the runtime equality with a base-class presence check + keep the `@ts-expect-error` as the type guard. (Fixed inline — Task 5 Step 3.)
