Playwright e2e walked the staff-critical loop against the dev deployment: QRIS sale, BCA VA sale, refund (mgr PIN), voucher online + offline, spoilage — **7 passed, 1 pre-existing skip**. Combined with the unit gate (typecheck ✅, lint 0 errors, vitest 1008/1008), this confirms the static-audit fixes in-browser. Note: the e2e simulate helper needs `XENDIT_SECRET_KEY` exported in the shell (`npx convex env get XENDIT_SECRET_KEY`) — without it the 4 payment-flow specs fail at the simulate step, not a code regression.
